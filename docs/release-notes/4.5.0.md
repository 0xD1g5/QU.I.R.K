# QU.I.R.K. 4.5.0 — Reliability & Gap Closure

**Released:** 2026-05-03
**Milestone:** v4.5

## What's New

QU.I.R.K. 4.5 is a reliability and hardening release. Every scanner is hardened against
missing optional extras, connection timeouts, and unexpected exceptions. CBOM validation
is closed against the CycloneDX 1.6 schema. The dashboard reaches WCAG AA compliance.
The Data at Rest dashboard tab (deferred from Phase 27) ships. Seven phases address
accumulated technical debt across CI stability, chaos lab parity, and UAT automation.

### Identity API Regression Fix

Phase 38 restored SAML/OIDC entries in `identity_findings` that had silently stopped
appearing in the API response (regression introduced in Phase 36). The deferred SAML
scan-window test from Phase 24 is re-enabled and GREEN. The `wave_0_complete` gate
added in Phase 36 is flipped to `true`, closing the planning audit-trail gap that had
been carried forward since the 4.4 release.

### Data at Rest Dashboard Tab

Phase 39 ships the DASH-05 Data at Rest tab deferred from Phase 27. The new `/dar`
React route surfaces database, object storage, Kubernetes, and Vault findings in a
dedicated view with per-surface encrypted/unencrypted breakdowns. The tab uses the
same shadcn/ui + Tailwind primitives as the rest of the dashboard and degrades
gracefully on pre-4.3 scans with no DAR endpoints.

### Chaos Lab Parity

Phase 40 brings `lab.sh`, the lab README, and `expected_results_v4.md` up to full
v4.4 parity. The `all` command in `lab.sh` now derives the profile list dynamically
from `docker-compose.yml` — the profile inventory never drifts from the compose file.

### Scanner Robustness

Phase 41 hardens all scanners against missing optional extras, timeouts, and unexpected
exceptions. A consistent timeout/retry policy is documented in `docs/timeout-retry-audit.md`.
Every deferred test from code-reasons (non-live-infra) is now GREEN in CI.

### CBOM Correctness Audit

Phase 42 validates CBOM JSON and XML output against the CycloneDX 1.6 schema. Classifier
unknown-fallback gaps are closed. The Pass-2 and Pass-3 skip lists (which prevent hollow
`CertificateProperties` on non-TLS endpoints) are audited and unit-tested. A new
coverage report (`docs/cbom-classifier-coverage.md`) enumerates every algorithm surfaced
by `build_cbom()` against all 18 chaos lab profiles.

### Dashboard Polish

Phase 43 eliminates browser console errors and React warnings across all routes, adds
explicit loading and empty states to every view, and meets WCAG AA color contrast and
keyboard-navigation requirements. The Data at Rest tab empty-state and the motion tab
loading state are included.

### UAT Automation

Phase 44 automates Phase 27 (database), Phase 29 (Kubernetes), Phase 25 (identity),
and Phase 30 (Vault) UAT scenarios against existing chaos lab profiles with pytest.
All four UAT scenarios were previously manual-only.

## Upgrade Guidance

