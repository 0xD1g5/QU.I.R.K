# QU.I.R.K. 4.6.0 — Enterprise Readiness

**Released:** 2026-05-05
**Milestone:** v4.6

## What's New

QU.I.R.K. 4.6 closes the gap between a capable tool and an enterprise-deployable product.
Six phases deliver clean installation, actionable certificate findings, multi-target
workflows for large estates, plain-English risk context on every finding, structured
compliance framework mappings, and production-quality documentation. Delivered in 3 days
(2026-05-03 → 2026-05-05): 105 commits, 125 files changed, +20,560 / -405 lines.

### Install-Day UX

Phase 45 eliminates `ImportError` crashes on a clean `pip install quirk` or
`pip install quirk[all]`. A centralized optional-extra registry (`quirk.extras`) manages
the dependency-availability state for all optional scanners. When an extra is absent,
each scanner entry point emits a named advisory notice (logged at INFO level) rather than
raising an exception or silently skipping. The `[all]` meta-extra now correctly excludes
`impacket` on macOS Apple Silicon where it is unavailable.

### TLS Certificate Findings

Phase 46 adds explicit findings for four certificate defect classes that previously
produced zero findings even when clearly present:

| Finding | Severity | Trigger |
|---------|----------|---------|
| `CERT_EXPIRED` | HIGH | Certificate `notAfter` is in the past |
| `CERT_SELFSIGNED` | MEDIUM | Issuer equals subject |
| `CERT_UNTRUSTED_CA` | MEDIUM | Chain cannot be verified against trusted roots |
| `CERT_WEAK_KEY` / `CERT_RSA1024` | HIGH | RSA key < 2048 bits or EC key < 256 bits |

A `chain_verified` column is added to the `crypto_endpoints` table (idempotent `ALTER
TABLE` migration). A new `tls-cert-defects` chaos lab profile (ports 13444–13447) provides
four regression targets, one per finding class. The D-04 mutual exclusivity guard ensures
`CERT_EXPIRED` and `CERT_EXPIRING_SOON` are never emitted simultaneously for the same
endpoint.

### Nmap Discovery and Multi-Target Wizard

Phase 47 removes the biggest friction point for enterprise scans. Users can now pass:

```bash
quirk scan --targets 10.0.1.0/24
quirk scan --targets host1,host2,host3
quirk scan --targets-file targets.txt
```

Optional nmap-based port discovery (`--nmap-discover`) pre-populates the open-port list
for each target without manual enumeration. A probe-budget guard caps the nmap invocation
to a configurable `--max-parallelism` (default 100) to prevent flooding the network.
`nmap` is added to the optional-extra registry with a graceful advisory when absent.

A separate Phase 47 plan adds CycloneDX JSON schema validation as an optional post-scan
step: `quirk[validate]` installs `cyclonedx-schema`; validation failure emits an advisory
finding rather than crashing the run.

### Rich Finding Context

Phase 48 ensures every finding is actionable. Each finding now carries:

- A `description` field with a plain-English explanation of why the condition is a risk
- A `recommendation` field with a specific remediation step
- For quantum-relevant findings: the FIPS 203/204/205 algorithm to migrate to and the
  NIST IR 8547 deprecation deadline for the vulnerable algorithm

All pre-standardization terminology (`Kyber`, `Dilithium`, `Falcon`) is purged from the
codebase. A CI grep gate (`tests/test_pqc_terminology_gate.py`) enforces this permanently.

### Compliance Framework Mapping

Phase 49 adds a `quirk/compliance/` module that maps each finding type to machine-readable
control references. Three frameworks ship in v4.6:

| Framework | Controls Mapped |
|-----------|----------------|
| PCI DSS 4.0.1 | Requirement 4.2.1 (strong crypto in transit), 12.3.3 (crypto inventory) |
| HIPAA 45 CFR | § 164.312(a)(2)(iv) (encryption at rest), § 164.312(e)(2)(ii) (encryption in transit) |
| FIPS 140-3 | Module usage requirements — `approved` / `non-approved` classification |

Compliance references are attached to every `FindingItem` by `_build_finding()` and flow
through to the HTML/PDF report as a "Compliance Summary" section. The `quirk compliance
status` CLI command reports per-framework coverage and mapping freshness.

Each mapping entry includes `version`, `last_verified`, and `source_url` fields. A
`STALENESS_THRESHOLD_DAYS = 365` constant in the module triggers a CI warning when
mappings have not been reviewed within the cadence.

## Upgrade Guidance

```bash
pip install --upgrade quirk
```

The `[all]` meta-extra is the recommended install for new deployments:

```bash

```

fully forward-compatible. The new `chain_verified` column is added by an idempotent
`ALTER TABLE` on first run — no manual migration required.

## Known Limitations

- **`quirk[all]` on macOS Apple Silicon** excludes `impacket`. Kerberos scanning on Apple
  Silicon requires a manual `pip install impacket` under Rosetta or in a Linux container.
- **Nmap discovery requires nmap in PATH.** Advisory is emitted if absent; scanning falls
  back to the configured explicit target list.
- **CycloneDX schema validation is advisory.** `validate_schema: true` emits a finding on
  validation failure rather than blocking the scan run, to preserve the agentless/offline
  operational model.
