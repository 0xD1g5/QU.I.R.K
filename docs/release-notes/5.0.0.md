# QU.I.R.K. 5.0.0 — Stabilization + Tech Debt Sweep

**Released:** 2026-05-22
**Milestone:** v5.0

## What's New

QU.I.R.K. 5.0 is a stabilization and tech-debt-sweep release. No new scanning surfaces or
behavioral changes are introduced. The one demoable capability anchor is the OQS-nginx
PQC-hybrid chaos-lab profile, which establishes a concrete post-quantum agility ceiling in
the scoring engine. The remaining four phases clear dependency hygiene, scoring transparency,
chaos-lab coverage, and code cleanup work accumulated since v4.10.

### OQS-nginx PQC-hybrid Lab Profile and Scoring Ceiling (Phase 90)

The marquee addition in v5.0 is a digest-pinned
[openquantumsafe/nginx](https://hub.docker.com/r/openquantumsafe/nginx) chaos-lab profile
serving TLS 1.3 with the X25519MLKEM768 hybrid key encapsulation mechanism (KEM) and an
ML-DSA-65 certificate on port 39444.

Key properties of this profile:

- **Hybrid KEM (X25519MLKEM768):** Combines classical X25519 Diffie-Hellman with
  ML-KEM-768 (FIPS 203) for harvest-now-decrypt-later resistance.
- **ML-DSA-65 certificate:** Post-quantum digital signature (FIPS 204).
- **Live detection:** The QUIRK scanner runs a `openssl s_client -groups X25519MLKEM768`
  probe against port 39444. A successful negotiation increments
  `pqc_hybrid_endpoint_count` in the evidence dictionary.
- **Agility scoring bonus:** `agility_pqc_hybrid_bonus = 8.0` in `SCORE_WEIGHTS` — the
  highest individual agility signal — anchors the PQC scoring ceiling. An estate with at
  least one hybrid-KEM endpoint can reach agility 25/25.
- **Digest-pinned image:** `openquantumsafe/nginx@sha256:6ca18ac6...` (built 2026-05-18).
  `oqs-provider` renames group names across releases; pinning prevents silent behavior
  drift.
- **CHAOS-05 compliant:** Image pin verified by the existing pinning gate.

To bring up the lab profile:

```bash
PROFILE_ARGS="--profile oqs-nginx" ./lab.sh up
# verify negotiation
openssl s_client -connect localhost:39444 -groups X25519MLKEM768 2>&1 | grep "Negotiated"
```

### Scoring Transparency — Subscore Decomposition (Phase 88)

CLI markdown, executive markdown, and HTML/PDF reports now include a `Score Decomposition`
table showing all six subscore values (N/25) alongside the `sum ÷ 1.5 = total/100` rollup
formula. This makes the scoring arithmetic visible and auditable without requiring access
to the raw JSON evidence.

CBOM builder closes Phase 42 OBS-1: five profiles that previously emitted zero algorithm
components (database, registry, source, ssh-weak, storage-s3) now emit either real
algorithm components (where detectable) or affirmative `quirk:coverage-note` properties
explaining why no cryptographic material was observed.

Forward-locking tests lock both the orthogonal scoring model (single engine, no
cross-category leakage) and the render-parity contract (all three report surfaces produce
equivalent subscore blocks).

### Chaos Lab — Four New Weak-TLS Profiles (Phase 89)

Four standalone Docker Compose profiles add coverage for database, cache, message-broker,
and gRPC TLS posture:

| Profile | Port(s) | Weakness |
|---------|---------|---------|
| `postgres-tls` | 39432 | RSA-KX cipher suites (`AES128-SHA`, `AES256-SHA`) + TLS 1.2 only |
| `redis-tls` | 39380 (TLS), 39379 (plaintext) | `DES-CBC3-SHA` + plaintext port |
| `kafka-tls` | 39093 (TLS), 39092 (plaintext) | RSA-KX ciphers + plaintext KRaft listener |
| `grpc-tls` | 39443 | RSA-2048 key (Go TLS 1.2 defaults — modern ciphers but weak key size) |

The `grpc-tls` profile uses a multi-stage Go Dockerfile with the `grpc-go` library.
An empirical sslyze ALPN-h2 gate (D-03) confirmed that QUIRK's scanner handles gRPC's
`NextProtos: ["h2"]` constraint without requiring a special openssl fallback — standard
TLS cipher and certificate findings are emitted normally.

LAB-03 (SMTP STARTTLS standalone profile) is closed as already covered by the `email`
profile's port 30587 (Postfix submission).

To bring up any profile:

```bash
PROFILE_ARGS="--profile postgres-tls" ./lab.sh up
PROFILE_ARGS="--profile redis-tls" ./lab.sh up
PROFILE_ARGS="--profile kafka-tls" ./lab.sh up
PROFILE_ARGS="--profile grpc-tls" ./lab.sh up
```

### Dependency Hygiene (Phase 87)

- **Node 24 CI runtime:** Dashboard CI (`dashboard-quality.yml`) upgraded from Node 20 to
  Node 24 ahead of GitHub's 2026-06-16 forced switch. PR #4 demonstrated a clean
  end-to-end run (setup, `npm ci`, `tsc + vite build`, ESLint) on Node 24.
- **ESLint fixes:** Seven pre-existing ESLint errors (react-refresh, react-hooks,
  TypeScript `any`, unused directive) resolved in the same change.
- **lxml/XXE hardening:** `defusedxml` replaced by a single `quirk/util/xml_safe.py`
  chokepoint using `lxml.etree.XMLParser` with five explicit security flags
  (`resolve_entities=False`, `no_network=True`, `load_dtd=False`,
  `dtd_validation=False`, `huge_tree=False`). Both XML-consuming modules (`nmap_parser`,
  `saml_scanner`) route through this chokepoint. Six forward-locking CI tests enforce the
  XXE/billion-laughs protection contract. `defusedxml` removed from `pyproject.toml`.

## Upgrade Guidance

```bash
pip install --upgrade quirk-scanner
```

**No breaking changes.** All v4.10 configuration files, databases, and scan outputs are
forward-compatible. The new subscore decomposition block in reports is purely additive.
The new `agility_pqc_hybrid_bonus` weight is wired to the `pqc_hybrid_endpoint_count`
evidence key introduced in Phase 90; legacy scans without this key receive no bonus
(zero is the safe default).
