## Gaps (Actionable)

### Gap 1: CLI-04 — pyproject.toml version mismatch

**File:** `pyproject.toml` line 7
**Current:** `version = "4.0.0"`
**Required:** `version = "4.1.0"`
**Impact:** `pip show quirk` and `importlib.metadata.version("quirk")` return "4.0.0" for installed packages, contradicting runtime `__version__` = "4.1.0".
**Fix:** One-line change in pyproject.toml. No test currently catches this — consider adding a test that reads the package manifest.

### Gap 2: SCORE-04 — Interactive mode output dir diverges from dashboard default

**File:** `quirk/interactive.py` line 165
**Current:** `out_dir = _prompt("Output directory", "output")`
**Required default:** `"quirk-output"` (to match `QUIRK_OUTPUT_DIR` dashboard default)
**Impact:** Interactive-mode users who accept the default output directory will never see their selected scan profile reflected in the dashboard — the dashboard silently falls back to "balanced" profile because no intelligence JSON is found at "./quirk-output/".
**Fix:** Change interactive.py:165 default from `"output"` to `"quirk-output"` and update line 166 `db_path` default accordingly.
