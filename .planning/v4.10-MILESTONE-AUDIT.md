## Post-audit Decision: v4.10-D-06 (supersedes v4.10-D-05)

**Logged 2026-05-22, release-time.** During UAT-85-11 (manual tag-push gate), the PyPI Pending Trusted Publisher form rejected the distribution name `qu-i-r-k` with the literal error *"This project name is too similar to an existing project"* — PyPI's PEP 541 typosquat protection flagged it as confusingly close to the existing (squatted) `quirk` project.

**Resolution:** PyPI distribution name changed from `qu-i-r-k` to `quirk-scanner`. The new name was dry-run-accepted via the Pending Publisher form (screenshot confirmed Gate 5.5 pass). Sweep applied to 15 active production + planning files: `pyproject.toml`, `quirk/__init__.py`, `tests/test_version.py`, `Dockerfile`, `Formula/quirk.rb`, `README.md`, `docs/getting-started.md`, `docs/release-process.md`, `docs/UAT-SERIES.md`, `docs/upgrade-guide.md`, `.github/workflows/release.yml`, `.github/workflows/release-container.yml`, `.planning/PROJECT.md`, `.planning/ROADMAP.md`, `.planning/v4.10-MILESTONE-AUDIT.md` (this file). Archived milestone snapshots under `.planning/milestones/v4.10-*` left as-is — they reflect what was true at milestone close (`qu-i-r-k`) and the D-06 supersede records the post-milestone correction.

**Identifiers UNCHANGED by D-06:** CLI command (`quirk`), Python import name (`quirk`), GHCR image namespace (`ghcr.io/0xd1g5/quirk`), Homebrew tap repo (`0xD1g5/homebrew-quirk`), brew install command (`brew install 0xD1g5/quirk/quirk`). Only PyPI-side strings (`pip install quirk-scanner[all]`, `importlib.metadata.version("quirk-scanner")`, `https://pypi.org/p/quirk-scanner`) and `pyproject.toml [project.name]` changed.

**Reason for not anticipating this in Phase 84-01:** Phase 84-01's PyPI name verification used `pip index versions quirk` to check whether the natural name was taken (it was — squatter). It did NOT include a Pending-Publisher dry-run for the chosen alternate `qu-i-r-k`. PEP 541 similarity rejection is enforced at *registration* time, not at `pip index versions` time. **Lesson for v4.10.1 (BACK-90 / Phase 86): add a Pending-Publisher dry-run step to any future PyPI name verification.** The "namespace empty" check (`pip index versions`) is necessary but not sufficient — only the actual Pending-Publisher form acceptance proves the name is registerable.
