<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator JWT target list → fetch URL | Operator-supplied target URLs that may already carry a probed key param |
| operator --config path → scheduler | Operator-supplied config that must never persist credentials in scheduled scans (D-11) |
| in-memory secret → URL query string | api_key_query secret appended to the fetch URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-97-06 | Information Disclosure | _append_query_param reject message (D-03) | mitigate | The reject path logs via `safe_str(url)` only and must not interpolate the raw pre-existing param value or the secret. Test 2 asserts the message does not contain the pre-existing value. No new plaintext-secret surface is introduced. |
| T-97-07 | Tampering / Repudiation | one rejected target aborting the scan (D-03) | mitigate | The caller-continue requirement (Test 3) ensures a rejected target does not silently kill coverage of the remaining targets — preventing a single crafted target from suppressing the rest of the scan. |
| T-97-08 | Elevation of Privilege / fail-open | scheduler auth-reject (D-05 / D-11) | mitigate | This fix STRENGTHENS the D-11 fail-closed control: the extension gate (a bypass) is removed and the parse-failure branch is changed from fail-OPEN (return False, allow) to fail-CLOSED (return True, reject). Test 4 enforces fail-closed; Test 1 enforces extensionless detection. The control is tightened, not weakened. |
| T-97-09 | Denial of Service | yaml.safe_load on arbitrary existing file (D-05) | accept | yaml.safe_load (not full load) is already the in-file pattern and is the safe loader (no arbitrary object construction). Parse cost is bounded by the operator's own config file; the operator already controls --config. Removing the extension gate does not expand who can supply the path. No untrusted remote input reaches this parser. |
| T-97-10 | Tampering | misclassification regression (D-05) | mitigate | Tests 1-4 pin the four classification outcomes (auth→reject, none→allow, non-auth→allow, unparseable→reject) so a future change cannot silently restore the fail-open or extension-gate behavior. |
| T-97-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan; PyYAML is an existing base dependency. N/A — no new dependencies. |

**Control-weakening check:** D-03 adds a guard and keeps iteration safe (no new leak,
no abort); the reject message is safe_str-scrubbed. D-05 tightens a fail-closed control:
removes a known bypass (extension gate) and flips the parse-failure branch from fail-open
to fail-closed. yaml.safe_load is the safe loader and the path is already operator-controlled,
so broadening the parse introduces no new untrusted-input or parse-DoS exposure. No control
is loosened.
