# Phase 56 — UI Design Contract

> Visual and interaction contract for the QRAMM print section added to `/print` route.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** This phase adds a single new sub-component (`PrintQRAMM`) to the existing
> static-CSS print page (`print.tsx`). The interactive dashboard is NOT modified. All design

## Spacing Scale

Declared values (multiples of 4 only). Print route uses inline `style={{}}` or static CSS only.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Badge horizontal padding (1px top/bottom, 6px left/right — matches existing `.badge` rule) |
| sm | 8px | Table cell padding (5–6px vertical, 8px horizontal — matches existing `th`/`td` rules) |
| md | 16px | Section margin-bottom for framework sub-sections; body paragraph gaps |
| lg | 24px | `.print-section` padding-top (existing rule preserved); `h2` margin-top |
| xl | 32px | Score-row gap (existing `.score-row` rule uses `gap:32px`) |
| 2xl | 48px | Not used in print route |
| 3xl | 64px | Not used in print route |

Exceptions:
- Radar SVG: fixed `200×200` viewBox with `radius = 80` (compass positions). Polygon points
  computed as `score / 4 * 80` per axis. Container div: `width:200px; height:200px; margin:16px 0`.
- Table cell vertical padding: 5px (existing `td` rule) — not a multiple of 4. Preserved as-is
  to not regress existing sections.

```

  <h3>Compliance Framework Coverage</h3>
  [Executive summary table — 8 rows]
  [Per-framework detail — h3 + table × 8, flowing continuously]

```

```

  <h2>QRAMM Governance Assessment</h2>

    the dashboard to populate this section.</p>

```

The `<h2>` heading always renders. The body varies between scored and no-session states (D-05).

## Radar SVG Specification

| Property | Value |
|----------|-------|
| viewBox | `0 0 200 200` |
| Center | `(100, 100)` |
| Radius | `80` (max axis length from center) |
| CVI axis | Top — `(100, 20)` endpoint |
| SGRM axis | Right — `(180, 100)` endpoint |
| DPE axis | Bottom — `(100, 180)` endpoint |
| ITR axis | Left — `(20, 100)` endpoint |
| Axis lines | `<line>` from `(100,100)` to each endpoint, `stroke:#e4e4e7`, `stroke-width:1` |
| Score polygon | `<polygon>` with 4 computed points, `fill:rgba(75,168,168,0.18)`, `stroke:#4ba8a8`, `stroke-width:2` |
| Point computation | `cx = 100 + cos(angle) * (score/4 * 80)`, `cy = 100 + sin(angle) * (score/4 * 80)` per axis |
| Axis labels | `<text>` at each endpoint — dimension name (e.g., "CVI") at offset 10px beyond endpoint; `font-size="12"` |
| Score values | `<text>` adjacent to each polygon vertex — score to 1 decimal place; `font-size="12"` `font-weight="600"` |
| Container | `<div style="width:200px;height:200px;margin:16px 0">` wrapping the SVG |

## States and Interactions

This is a print route — no interactive elements beyond data loading. State matrix:

| State | Trigger | Visual |
|-------|---------|--------|
| Loading (both hooks pending) | Page load, either hook `loading=true` | Existing "Loading scan data..." div; `data-ready` NOT set on body |
| Loaded — scored session | Both hooks resolved, `scoreResult !== null` | Full QRAMM section: SVG radar + dimension table + compliance tables |
| Loaded — no session | Both hooks resolved, `scoreResult === null` | QRAMM section heading + no-session placeholder paragraph |
| Error (scan data) | `useScanData` returns error | Existing red error div; page renders nothing else |
| Error (QRAMM hook) | `useQRAMMPrintData` returns error | Log to console; render QRAMM section as no-session state (graceful degradation — do not block full PDF) |

**`data-ready` gate:** `data-ready` is set on `document.body` only when BOTH `useScanData.loading === false` AND `useQRAMMPrintData.loading === false`. This gates headless PDF capture correctly (CONTEXT.md code_context).
