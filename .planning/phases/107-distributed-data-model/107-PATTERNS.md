#### `init_db` orchestration pattern (lines 369-404)

```python
def init_db(db_path: str) -> Engine:
    engine = get_engine(db_path)

    with engine.connect() as conn:
        conn.commit()

    Base.metadata.create_all(engine, checkfirst=True)
    for table, columns in _ADDITIVE_MIGRATIONS:
        if table == "qramm_answers":
            continue  # handled after _ensure_qramm_tables below
        _ensure_columns(engine, table, columns)
    _ensure_qramm_tables(engine)
    _ensure_columns(engine, "qramm_answers", _PHASE54_QRAMM_ANSWER_COLUMNS)
    _ensure_qramm_profiles_fk(engine)
    _ensure_scheduled_tables(engine)
    _ensure_scan_jobs_table(engine)
    _ensure_scan_checkpoints_table(engine)
    _ensure_integration_deliveries_table(engine)
    return engine
```

The new `CREATE INDEX IF NOT EXISTS` step for `crypto_endpoints.sensor_id` is inserted **after** the `_ADDITIVE_MIGRATIONS` loop (so the column exists) and **before** the `return engine`. Use `engine.connect()` + `conn.execute(text(...))` + `conn.commit()` — the same pattern as `_ensure_columns` uses internally. The idempotent `IF NOT EXISTS` clause makes it safe to call repeatedly.

Exact DDL shape to use (satisfies allowlist spirit, not `_SAFE_COL_TYPE_RE` which gates columns not index names):

```python
    with engine.connect() as conn:
        conn.execute(text(
            "CREATE INDEX IF NOT EXISTS ix_crypto_endpoints_sensor_id"
            " ON crypto_endpoints (sensor_id)"
        ))
        conn.commit()
```

```python
def _column_snapshot(engine) -> dict:
    """Return a dict mapping table_name -> sorted list of column names."""
    insp = sa_inspect(engine)
    return {
        t: sorted(c["name"] for c in insp.get_columns(t))
        for t in insp.get_table_names()
    }
```

## Shared Patterns

### FK enforcement listener (already active — no new code needed)

**Source:** `quirk/db.py` lines 31-44

```python
@event.listens_for(Engine, "connect")
def _sqlite_fk_pragma(dbapi_connection, connection_record):
    cursor = dbapi_connection.cursor()
    try:
        cursor.execute("PRAGMA foreign_keys=ON")
    finally:
        cursor.close()
```

This fires for every Engine in the process, including in-memory test engines. The `ON DELETE CASCADE` FKs on `sensor_tokens` and `sensor_pushes` will be enforced automatically — no new wiring needed. Tests that verify cascade behavior must use a real on-disk SQLite (or `create_engine("sqlite:///...")`) — NOT `sqlite://` (in-memory) unless the test explicitly creates the schema after the engine is created.

### Allowlist guards (must not bypass)

**Source:** `quirk/db.py` lines 47-52

```python
_SAFE_COL_RE      = re.compile(r"^[a-z][a-z0-9_]*$")
_SAFE_COL_TYPE_RE = re.compile(r"^(TEXT|INTEGER|REAL|BOOLEAN|DATETIME|VARCHAR\(\d{1,4}\))$")
```

Every DDL fragment in `_V54_SENSOR_COLUMNS` must pass `_SAFE_COL_TYPE_RE`. Acceptable types for the new columns: `TEXT` (for `sensor_id`, `segment`). The `CREATE INDEX IF NOT EXISTS` step bypasses `_ensure_columns` and is written directly with `text(...)` — it does not need to pass `_SAFE_COL_TYPE_RE` (that regex gates column type fragments, not index DDL), but the index name and table/column names must be hardcoded string literals (no user-controlled interpolation).

### Token hashing (SHA-256, never store raw)

**Source:** `quirk/cli/token_cmd.py` line 100

```python
token = secrets.token_urlsafe(32)
```

The raw token is printed once and discarded. `sensor_tokens` stores `hashlib.sha256(token.encode()).hexdigest()` — the hash, never the token. This mirrors the Phase 102 dashboard auth pattern (CONTEXT D-02 from Phase 106). No code exists yet in the codebase that does this hashing — Phase 107 adds the schema only; the hashing logic lands in Phase 108 (enrollment CLI). The `token_urlsafe` call is the upstream source to reference in the Phase 108 analog search.

### `tmp_path` pytest fixture (preferred over `tempfile.NamedTemporaryFile`)

**Source:** `tests/test_init_db_idempotent.py` — all test functions use `tmp_path: Path`
