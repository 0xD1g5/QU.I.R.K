**Step 2 — Fix `tests/test_saml_scanner.py`:**

a. Inspect the existing top-of-file imports. If only `import datetime` is present, ADD a single import line directly under it:
   ```python
   from datetime import timezone as _tz
   ```
   If `from datetime import timezone` already exists, reuse it (no new import).

b. Replace every occurrence of `datetime.datetime.utcnow()` (and any module-aliased variant) with `datetime.datetime.now(_tz.utc).replace(tzinfo=None)`.

   Concrete example (before):
   ```python
   .not_valid_before(datetime.datetime.utcnow())
   .not_valid_after(datetime.datetime.utcnow() + datetime.timedelta(days=365))
   ```

   After:
   ```python
   .not_valid_before(datetime.datetime.now(_tz.utc).replace(tzinfo=None))
   .not_valid_after((datetime.datetime.now(_tz.utc) + datetime.timedelta(days=365)).replace(tzinfo=None))
   ```

   Use `.replace(tzinfo=None)` because the `cryptography` library's
   `not_valid_before`/`not_valid_after` historically required naive
   datetimes; the strip is safe on aware datetimes that originated from
   `now(_tz.utc)` and preserves equivalent UTC time semantics for cert
   validity checks. Newer `cryptography` versions accept aware datetimes
   too, so this remains forward-compatible.

c. Do not modify any other content. Keep diff minimal (per CLAUDE.md).
