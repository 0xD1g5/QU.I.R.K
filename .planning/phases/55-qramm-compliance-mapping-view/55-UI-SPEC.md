## Layout Contract

### 6th Tab: Compliance Map

Tab trigger label: `Compliance Map` (matches `value="compliance"` — no collision with existing values `cvi`, `sgrm`, `dpe`, `itr`, `scorecard`)

Tab content root: `<div className="space-y-6 pt-4">` — identical to ScorecardTab root

### Content Structure (scored state)

```
[Card: 8-Framework Coverage Table]
  CardHeader: "Compliance Framework Coverage"
  CardContent:
    Table
      TableHeader: Practice Area | Dimension | NIST PQC | NSM-10 | CNSA 2.0 | ISO 27001 | ETSI QS | PCI-DSS | CC | BSI TR
      TableBody: one row per practice area (12 rows)
        - Practice area cell: text-sm (e.g. "1.1 — Cryptographic Inventory")
        - Dimension cell: text-xs label-eyebrow (e.g. "CVI")
        - Per-framework cells: font-data relevance score (0.00–1.00) or "—" when null
    [Footnote paragraph below table]
      text-xs text-muted-foreground
      "Coverage reflects QUIRK scanner findings for CVI only — SGRM, DPE, ITR require manual assessment."

[Card: Coverage Tier Legend]
  CardContent: two Badge examples with labels (inline, horizontal)
    [Scanner-informed badge] — "At least one relevant dimension is CVI-informed"
    [Manual only badge]      — "All relevant dimensions require manual assessment"
```

### Content Structure (unscored state / no session)

```
[Card: Unscored Banner]
  CardContent:
    Banner text (text-sm text-muted-foreground):
    "Run and score a QRAMM assessment to see session-derived relevance scores."
    
[Card: 8-Framework Coverage Table — static weights only]
  (same table structure as scored state, all relevance_score cells show "—")
  (footnote visible)
```

### Column Layout for the Coverage Table

The table groups per-framework scores as columns. Each of the 8 framework columns shows the `relevance_score` value for that row's practice area, formatted to 2 decimal places using `.font-data`. When `relevance_score` is null, render `—` (em dash, not a zero).

The `scanner_informed` boolean from the API drives the framework-level badge in the legend only — it is NOT rendered inline in every table row (that would create 96 badge instances, too dense). The coverage tier badges appear once per framework in the legend card below the table.

Source: `55-CONTEXT.md` D-06, D-08

## Interaction Contract

### Fetch Trigger
- Fetch `GET /api/qramm/sessions/{sessionId}/compliance-map` when the tab mounts (`useEffect` with `[ctx.sessionId, ctx.scoreResult]` deps)
- Show `Loader2` spinner (24px, centered) while request is in flight
- On success: render table
- On network error: show inline error state (see Copywriting Contract)
- No manual refresh button — data re-fetches automatically when `ctx.scoreResult` changes (i.e., after Calculate Score)

### Loading State
- Centered `<Loader2 className="animate-spin h-6 w-6 text-muted-foreground" />` inside the table card's CardContent area
- Minimum height `min-h-[200px]` on CardContent to prevent layout shift

### No destructive actions in this tab. No confirmation dialogs required.

Source: ScorecardTab.tsx async fetch pattern; `55-CONTEXT.md` D-02, D-03

## Accessibility Contract

- Table must have a visible `<caption>` or `aria-label` on the `<Table>` element: `aria-label="QRAMM compliance framework coverage by practice area"`
- "Scanner-informed" and "Manual only" badges must carry `title` attributes when rendered as interactive chips without surrounding label text
- Color must never be the sole differentiator between badge states — the text label is the primary indicator, color is reinforcement only
- Loading spinner: `aria-label="Loading compliance map"` on the `Loader2` wrapper div
