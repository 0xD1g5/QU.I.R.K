evaluator, and a FastAPI BackgroundTask that runs the existing `merge_scan()`

deployment case (AUTOMERGE-01) while keeping merge failure structurally isolated

|----------|-------------|
| sensor → console push | Authenticated via require_sensor_auth (Phase 113); revoked tokens rejected before sensor_push runs |
| background task → DB | Auto-merge opens its own session post-response; merge failure must not touch the committed push txn |
| config.yaml → trigger eval | Operator-controlled file read per push; values must be validated before use |
