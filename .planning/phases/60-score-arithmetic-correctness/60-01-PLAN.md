- **file**: `quirk/intelligence/scoring.py`
- **action**: At line 219, replace the unclamped addition with a `_clamp()` call. The `_clamp()` helper already exists at line 65.

### Task 01-03: Replace Pydantic multiplier constraint with explicit 400 guard (SCORE-02)

- **type**: implementation
- **file**: `quirk/dashboard/api/routes/qramm.py`
- **action**: Two changes in this file.

  **Change A — ScoreRequest model (line 93):** Remove the `ge=0.8, le=1.5` Pydantic field constraints so out-of-range values are not rejected at serialisation with a 422 before reaching the route handler.

  Before (line 93):
  ```python
  profile_multiplier: Optional[float] = Field(default=None, ge=0.8, le=1.5)
  ```

  After (line 93):
  ```python
  profile_multiplier: Optional[float] = Field(
      default=None,
      description="Profile risk multiplier. Valid range: [0.8, 1.5]. "
                  "Values outside this range return HTTP 400 (QRAMM_MULTIPLIER_OUT_OF_RANGE).",
  )
  ```

  **Change B — score_session() route (after line 337, before the DB query at line 340):** Insert the explicit guard immediately after the `multiplier` extraction at line 337.

  Insert after line 337 (after `multiplier = ...`):
  ```python
  if payload is not None and payload.profile_multiplier is not None:
      if not (0.8 <= multiplier <= 1.5):
          raise HTTPException(
              status_code=400,
              detail={
                  "error_code": "QRAMM_MULTIPLIER_OUT_OF_RANGE",
                  "message": "profile_multiplier must be in [0.8, 1.5]",
                  "valid_range": [0.8, 1.5],
              },
          )
  ```

  Per D-04 and D-05: values inside `[0.8, 1.5]` are accepted and passed through unchanged. The Pydantic docstring (Change A) satisfies D-06 for OpenAPI schema documentation.

- **verify**: Start the dev server (`uvicorn quirk.dashboard.api.main:app --port 8765 &`) and run:

  Expect `400`. Then `curl ... -d '{"profile_multiplier": 1.2}'` — expect `404` (session does not exist, not 400/422).
  Or run the unit test from Plan 60-02: `pytest tests/test_qramm_multiplier.py -x`.
- **acceptance_criteria**: POST with `profile_multiplier=2.0` returns HTTP 400 with `error_code: QRAMM_MULTIPLIER_OUT_OF_RANGE`; POST with `profile_multiplier=1.2` does not return 400 (returns 422 for empty session or proceeds normally)
