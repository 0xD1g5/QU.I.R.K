<objective>
Fix Bug 3 (vault-seed `rsa-1024` rejection, profile: storage — deprecated) by removing the deprecated `storage` profile entirely, per CONTEXT D-3 recommendation. The profile is a v4.1 leftover whose three concerns were split into clean v4.3 successors:
- LocalStack KMS (port 20007) — was config-introspection-only, never a scanner protocol target
- Vault legacy 1.15 (port 20009) — replaced by `vault` profile's vault-30 (port 28200, image 1.17)
- postgres-pgcrypto (port 20010) — encryption-at-rest concerns covered by `database` profile's postgres-ssl-off (port 25432)

Root cause (v4.8-ROADMAP.md BACK-90 row): `storage/vault-seed.sh` line 19 attempts `vault write -f transit/keys/rsa-1024 type=rsa-1024` — Vault transit engine doesn't accept `rsa-1024`. Log evidence: `Code: 400. * unknown key type rsa-1024`.

Decision (CONTEXT D-3 recommended option — chosen): **Remove the deprecated `storage` profile entirely.** Rationale (3 sentences):
1. CONTEXT explicitly recommends this; README.md row 33 already marks the profile `**deprecated** — see database / storage-s3 / vault`; the bug exists only in code seeding a profile that should be retired.
2. v4.3 successor profiles already provide cleaner per-resource protocol coverage; fixing the rsa-1024 bug would extend the lifetime of dead code with no functional benefit.
3. Simpler lab = fewer drift bugs going forward; this phase is about closing drift, not preserving it.

This plan deletes the compose service blocks and orphaned seed scripts. README + oracle deletions are owned by Plan 05 (cross-cutting docs work) so that the cross-cutting commit captures all doc updates in one atomic change (image-pin policy note + README rows for bugs 1/2/3/4 + oracle deltas + UAT).

Purpose: Eliminate the bug by eliminating the deprecated code path.
Output: One compose YAML deletion, three seed-script deletions via `git rm`, one atomic commit `fix(chaos-lab):` prefix.
</objective>
