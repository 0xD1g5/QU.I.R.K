**7. Private scan function `_scan_kms(service, project_id: str, logger) -> List[CryptoEndpoint]`:**
- Build `project_resource = f"projects/{project_id}"`
- Auto-discover locations via `service.projects().locations().list(name=project_resource)` with `list_next()` pagination (per D-05)
- For each location, list key rings via `service.projects().locations().keyRings().list(parent=f"{project_resource}/locations/{location_id}")` with pagination
- For each key ring, list crypto keys via `service.projects().locations().keyRings().cryptoKeys().list(parent=key_ring_name)` with pagination
- Per D-04: enumerate primary version only. Extract algorithm: `primary = key.get("primary") or {}` then `algorithm = primary.get("algorithm", "")`. CRITICAL: guard against missing `primary` field (Pitfall 3). Also check `key.get("versionTemplate", {}).get("algorithm", "")` as fallback for asymmetric keys.
- Skip if primary state is not `"ENABLED"` -- check `primary.get("state", "")`.
- Look up algorithm in `GCP_KMS_ALGORITHM_MAP`. If found, set `cert_pubkey_alg = alg_name`, `cert_pubkey_size = key_size`.
- Protection level: `primary.get("protectionLevel", "SOFTWARE")` -> look up in `_PROTECTION_LEVEL_MAP`, default to `"CloudKMS/SOFTWARE"`.
- `cloud_scan_json=json.dumps({"gcp_algorithm": algorithm, "key_size": key_size, "protectionLevel": protection_level, "purpose": key.get("purpose", "")}, default=str)`
- Construct CryptoEndpoint: `host=key_name` (full resource name), `port=0`, `protocol="GCP"`, `service_detail=protection_level_string`.
- Outer try/except wraps the full function; inner try/except wraps per-key processing. Use `logger.v()` for errors.

**8. Private scan function `_scan_cloud_sql(service, project_id: str, logger) -> List[CryptoEndpoint]`:**
- List instances via `service.instances().list(project=project_id)` with `list_next()` pagination.
- Per D-08: extract `sslMode` from `instance.get("settings", {}).get("ipConfiguration", {}).get("sslMode")`.
- Treat `None`, `""`, and `"SSL_MODE_UNSPECIFIED"` as equivalent to `"ALLOW_UNENCRYPTED_AND_ENCRYPTED"` (Pitfall 7).
- If sslMode is `"TRUSTED_CLIENT_CERTIFICATE_REQUIRED"` -- skip (no finding, mTLS enforced).
- Otherwise look up in `SSL_FINDING_MAP`. If found, create CryptoEndpoint with:
  - `host=f"gcp://{project_id}/sql/{instance_name}"`
  - `port=0`
  - `protocol="CLOUD_SQL"`
  - `cert_pubkey_alg=severity` (e.g., "HIGH" or "MEDIUM")
  - `service_detail=instance_name`
  - `cloud_scan_json=json.dumps({"sslMode": ssl_mode, "finding": description}, default=str)`
- Outer/inner try/except with `logger.v()`.

**9. Private scan function `_scan_gcs(service, project_id: str, logger) -> List[CryptoEndpoint]`:**
- List buckets via `service.buckets().list(project=project_id)` with `list_next()` pagination.
- CRITICAL: response field is `"items"`, NOT `"buckets"` (Pitfall 4).
- Accumulate all bucket dicts in a `bucket_list: list`.
- After enumeration, create ONE sentinel endpoint per D-03:
  - `host=f"gcp://{project_id}/storage"`, `port=0`, `protocol="GCP"`, `cert_pubkey_alg="GCS-SUMMARY"`, `gcs_scan_json=json.dumps(bucket_list, default=str)`, `service_detail="GCS"`
- Then create per-bucket endpoints for findings:
  - Extract `encryption.defaultKmsKeyName` from each bucket dict.
  - If present -> `cert_pubkey_alg="CMEK"`. If absent -> `cert_pubkey_alg="Google-Managed"`.
  - `host=f"gcp://{project_id}/buckets/{bucket_name}"`, `port=0`, `protocol="GCP"`, `cloud_scan_json=json.dumps(bucket_dict, default=str)`, `service_detail="GCS"`
- Outer/inner try/except with `logger.v()`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config.yaml -> scan_gcp_targets | gcp_project_id crosses from config into API calls |

| gcp_connector.py -> CryptoEndpoint.scan_error | Exception messages from google.auth may contain env details |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-03 | Tampering | gcp_project_id input | mitigate | Validate non-empty string before any API call in scan_gcp_targets(); empty/None returns [] with logger warning |
| T-26-04 | Information Disclosure | DefaultCredentialsError in scan_error | mitigate | Store exception message in scan_error (visible in DB only, not in API responses); never log credential objects; catch at google.auth.default() and each execute() call |
| T-26-05 | Denial of Service | Large project with many locations/keyrings | accept | Auto-discovery may be slow on projects with 100+ locations; acceptable for scanner use case; gcp_kms_locations filter deferred per CONTEXT.md |
| T-26-06 | Spoofing | GCP API responses | accept | Scanner runs with ADC; API responses authenticated by google-auth TLS; no additional verification needed |
| T-26-07 | Elevation of Privilege | Connector API permissions | mitigate | Connector only calls read-only list APIs (cloudkms.cryptoKeys.list, sqladmin.instances.list, storage.buckets.list); document minimum required IAM roles in module docstring |
</threat_model>
