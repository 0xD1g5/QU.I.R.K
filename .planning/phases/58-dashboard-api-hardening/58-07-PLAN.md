```markdown
### Phase 58 — Dashboard API Hardening

**UAT-58-01: Bearer token auth (HARDEN-API-01)**
- Set QUIRK_API_TOKEN=test-token, start the dashboard
- Make a GET request to /api/qramm/sessions without Authorization header → expect 401
- Make a GET request with Authorization: Bearer test-token → expect 200
- With no QUIRK_API_TOKEN set → all requests pass through (auth disabled)
Pass criteria: 401 body = {"detail": "Authentication required"}

**UAT-58-02: CSRF header enforcement (HARDEN-API-01)**
- Set QUIRK_API_TOKEN=test-token, start the dashboard
- POST /api/qramm/sessions with valid Bearer token but without X-Quirk-Request header → expect 403
- POST with both Bearer token and X-Quirk-Request: 1 → expect non-403 response
Pass criteria: 403 body contains "Missing CSRF header: X-Quirk-Request"

**UAT-58-03: CORS allowlist (HARDEN-API-02)**
- Send OPTIONS preflight with Origin: http://evil.example.com → preflight returns no CORS headers (blocked)
- Send OPTIONS preflight with Origin: http://127.0.0.1:8512 → preflight returns CORS headers (allowed)
Pass criteria: Non-allowlisted origin receives no Access-Control-Allow-Origin header

**UAT-58-04: Rate limiting (HARDEN-API-03)**
- Send 61 POST requests to /api/qramm/sessions within 60 seconds (with valid auth + CSRF)
- 61st request → expect 429 with Retry-After header
- GET /api/health — send 100 GET requests → none return 429
Pass criteria: 429 with Retry-After; health always 200

**UAT-58-05: quirk init path-traversal guard (HARDEN-API-04)**
- Run: quirk init --output ../evil.yaml → expect warning message, no file written
- Run: quirk init --output /tmp/evil.yaml → expect warning message, no file written
- Run: quirk init --output quirk-output.yaml → expect normal config creation
Pass criteria: traversal paths print warning containing "outside" or "traversal"; no file created at malicious path

**UAT-58-06: PDF SSRF port guard (HARDEN-API-05)**
- Set QUIRK_SERVE_PORT=80 and call POST /api/export/pdf → expect 500 with {"detail": "QUIRK_SERVE_PORT is out of allowed range (1024–65535)."}
- Set QUIRK_SERVE_PORT=0 → same 500 response
- Normal operation (QUIRK_SERVE_PORT=8512) → PDF export succeeds

**UAT-58-07: @file target guard (HARDEN-API-06)**
- Run: quirk scan --targets @/etc/passwd → expect TargetFileError with reason path_not_allowed_prefix
- Create a file > 1 MB and run: quirk scan --targets @bigfile.txt → expect reason target_file_too_large
- Create a file with 10001 lines → expect reason target_file_too_many_lines
Pass criteria: each violation surfaces a clear error message; no silent failures
```

Update the `**Last Updated:**` date at the top of UAT-SERIES.md to 2026-05-09.
