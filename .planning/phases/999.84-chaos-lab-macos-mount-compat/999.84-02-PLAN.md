<objective>
Fix Bug 2 (DEF-999.83-B): `chaoslab-rabbitmq-broker-1` crashes on macOS Docker Desktop with `Error when reading /var/lib/rabbitmq/.erlang.cookie: eacces`. RabbitMQ requires the cookie file to be `0400 root:root`, and macOS Docker Desktop's bind-mount uid/gid mapping makes that permission unreachable.

Per CONTEXT D-Bug2 (recommended option (a)): set `RABBITMQ_ERLANG_COOKIE` as an env var. RabbitMQ honors the env value and never reads the file, so the bind-mount permission issue becomes moot. Single-line YAML change.

Purpose: Restore `broker` profile to healthy state on macOS without volume changes, custom Dockerfile, or AMQP/TLS config disturbance.
Output: One atomic commit `fix(chaos-lab): rabbitmq-broker macOS-host compat via RABBITMQ_ERLANG_COOKIE (BACK-91 bug 2)`.
</objective>
