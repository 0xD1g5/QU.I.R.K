## Change

`quantum-chaos-enterprise-lab/docker-compose.yml` — single line added to the rabbitmq-broker `environment:` list:

```diff
   environment:
     - RABBITMQ_CONFIG_FILE=/etc/rabbitmq/rabbitmq.conf
+    - RABBITMQ_ERLANG_COOKIE=lab-erlang-cookie-do-not-use-in-prod
   volumes:
```

Per CONTEXT D-2 option (a): when `RABBITMQ_ERLANG_COOKIE` env is set, RabbitMQ uses the env value directly and never tries to read `/var/lib/rabbitmq/.erlang.cookie` from the bind-mount path that macOS Docker Desktop cannot permission-match (the file requires 0400 root:root, which the bind-mount uid/gid mapping breaks).

Cookie value is the literal string `lab-erlang-cookie-do-not-use-in-prod` — fixed lab value, intentionally NOT a real secret. RabbitMQ's cookie is only used for inter-node Erlang clustering (irrelevant for a single-node chaos lab broker).
