## Change

`quantum-chaos-enterprise-lab/source/seed.sh` — added `repo_exists()` helper, gated each of the 3 `POST /api/v1/user/repos` calls behind `if repo_exists ...; then skip; else POST; fi`, and appended `|| true` to the `put_file` helper's curl call.

Per CONTEXT D-3 mixed strategy:
- **Repos (option b — existence check)**: `repo_exists "name"` does `GET /api/v1/repos/labadmin/{name}` and returns 0 if 200, non-zero if 404. The 3 repo-creation sites became `if repo_exists ...; then echo "Repo exists, skipping create"; else <original POST>; fi`. Preserves real-error visibility — auth failures or network issues still propagate via `curl -sf`.
- **Per-file puts (option a — `|| true`)**: the `put_file` helper appends `|| true` to its single curl call. In this seed context a 409 truly means "same path already committed with identical content" — the seed is fully deterministic in both content and commit messages — so swallowing it is safe and avoids doubling the API call count for ~10 file-puts.
