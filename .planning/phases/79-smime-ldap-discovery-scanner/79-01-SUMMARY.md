## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] LDIF `;binary` suboption rejected by OpenLDAP**
- **Found during:** Task 2 (first `./lab.sh up --profile smime` run)
- **Issue:** The plan + RESEARCH specified `userSMIMECertificate;binary:: <base64>` syntax (per RFC 2849 / RFC 4523), but OpenLDAP rejects this with `ldap_add: Undefined attribute type (17) — userSMIMECertificate;binary: option "binary" not supported with type`. The attribute (OID 2.16.840.1.113730.3.1.40) is defined with `SYNTAX 1.3.6.1.4.1.1466.115.121.1.5` (Binary), which already carries octets directly — the `;binary` transfer-encoding suboption is correct for `userCertificate` (LDAP syntax Certificate) but **not** for `userSMIMECertificate`.
- **Fix:** Updated `regen.sh` to emit `userSMIMECertificate:: <base64>` (no `;binary` suboption); added an explanatory comment in the LDIF preamble and in `expected_results_v4.md` so the next reader understands why we deviate from the must_have wording.
- **Files modified:** `quantum-chaos-enterprise-lab/smime/certs/regen.sh`, `quantum-chaos-enterprise-lab/smime/ldif/users.ldif` (regenerated), `quantum-chaos-enterprise-lab/expected_results_v4.md` (oracle note).
- **Commit:** `520b486`

**2. [Rule 1 — Bug] Seed sidecar exits 68 on re-run despite `ldapadd -c`**
- **Found during:** Task 2 second-run idempotency check
- **Issue:** `ldapadd -c` continues past already-exists errors on subsequent runs, but its exit code is still the **last** per-entry error code (68 = `LDAP_ALREADY_EXISTS`). This leaves the seed sidecar in a non-zero exit state on every idempotent re-run — visible in `docker ps -a` and any CI / monitoring system that watches sidecar exit codes. Must_have #4 ("Re-running `./lab.sh up --profile smime` does not error") was technically met at the `lab.sh` level (it doesn't wait on sidecars), but the sidecar itself looked broken.
- **Fix:** Wrapped the seed command: `... -f /ldif/users.ldif; rc=$?; if [ $rc -eq 0 ] || [ $rc -eq 68 ]; then exit 0; else exit $rc; fi`. Other failure modes (network errors, bad credentials, schema violations) still surface as non-zero exits.
- **Files modified:** `quantum-chaos-enterprise-lab/docker-compose.yml` (smime-seed `command:`).
- **Commit:** `520b486`
