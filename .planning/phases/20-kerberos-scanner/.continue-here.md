<decisions_made>

- **Per-etype CryptoEndpoint**: One endpoint per etype per host (mirrors DNSSEC per-zone-per-algorithm pattern) — each etype is a distinct finding row with its own severity
- **TCP primary / UDP fallback**: `_probe_kdc` uses impacket sendReceive (TCP); `_probe_kdc_udp` uses raw SOCK_DGRAM. TCP is tried first; UDP only on TCP failure
- **Test mocking strategy**: Mock `_probe_kdc`, `_probe_kdc_udp`, `_probe_ldap_anon` directly — avoids impacket dependency in dev env, cleaner than mocking raw impacket internals
- **kerberos-no-preauth placeholder**: When KDC returns AS-REP (no PREAUTH_REQUIRED), a single placeholder endpoint is emitted to preserve DB record
- **AES-128 is HIGH not SAFE**: Grover's algorithm halves effective key length to ~64 bits; AES-256 earns NIST level 1 (SAFE)
- **kerberos_scan_json**: `ldap_status` at top level AND under `ldap` key for compatibility with tests and future dashboard
</decisions_made>
