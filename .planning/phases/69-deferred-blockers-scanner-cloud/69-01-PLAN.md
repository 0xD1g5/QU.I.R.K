<objective>
Close BLOCK-01 (CR-07, CR-08 protocol): two narrow but real resource leaks.
1. `_scan_one_sslyze` does not deterministically clean up the sslyze `Scanner` object if `get_results()` raises mid-scan; sslyze owns an internal nassl thread/process pool released only at GC.
2. `fingerprint.py` opens a socket via `_tcp_connect` and only closes it via `with s:` on the next line; KeyboardInterrupt / SystemExit between the two statements leaks the socket.

Per D-05/D-06/D-07: wrap sslyze in try/finally with `del scanner` (+ optional `gc.collect()`); wrap the fingerprint socket span in `try/except BaseException: s.close()`; tests use `unittest.mock.patch` / `monkeypatch` to inject the exceptions and assert close.

Purpose: Hardens scanner against handle exhaustion under pathological conditions (long scans, Ctrl-C).
Output: try/finally guards in two scanner files; two new pytest files asserting cleanup.
</objective>
