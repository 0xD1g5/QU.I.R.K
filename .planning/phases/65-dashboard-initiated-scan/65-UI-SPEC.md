Declared values (all multiples of 4 — standard 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Compact element spacing, form field gap-y |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form group gap |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top padding |

- Sidebar width: 240px (lg breakpoint), 48px (icon-only, below lg) — inherited from `sidebar.tsx`, not overridden.

## Interaction Contract

### Form Validation

- Client-side: validate targets field non-empty before POST. Show inline error immediately on submit attempt without waiting for API.
- Server-side: display 422 field error messages inline, below the affected field. Do not clear the form on error.
- On 201: navigate immediately to `/scan/job/:jobId`. No intermediate success state.
- Button: show loading state (`disabled + spinner icon`) while POST is in flight. Re-enable on error.

### Polling Behavior

- Poll interval: 3000ms exactly (per CONTEXT.md D-07).
- Cancellation-safe: `let cancelled = false` + `return () => { cancelled = true }` pattern (per CONTEXT.md D-07, Phase 62 mandate).
- Stop polling on terminal status: `completed | failed | cancelled`.
- Auto-navigate on `completed`: call `setSelectedScanId(scan_run_id)` then `navigate("/")`.

### Stage Indicator

- 7 steps rendered as horizontal dots with labels below (visible on desktop; abbreviate to "N/7" on mobile below 640px).
- Completed steps: `--ds-ok` teal dot with checkmark icon (Lucide `CheckCircle2` 14px).
- Current step: `--primary` accent dot, animated pulse ring.
- Upcoming steps: `--ds-border` grey dot.
- Progress bar: `<Progress value={(stage_index / 7) * 100} />` — always visible, updates on each poll.

### Sidebar "New Scan" Button

- Position: top of sidebar, above all nav items, below the QU.I.R.K. wordmark.
- Style: `<Button variant="default" className="w-full">` on desktop, icon-only (`<Scan />` lucide icon, 16px) on mobile (below lg breakpoint).
- Not a nav link — does not receive the active-link highlight treatment.

### Accessibility

- All form fields have associated `<Label>` components with `htmlFor` matching `id`.
- Stage indicator: `role="list"` on the steps container; each step `role="listitem"` with `aria-label="Stage N: {name}, {status}"`.
- Progress bar: `aria-label="Scan progress"` + `aria-valuenow={stage_index}` + `aria-valuemax={7}`.
- Cancel button: `aria-label="Cancel scan job {jobId}"` on the destructive button.
- Loading state: `<PageSpinner ariaLabel="Loading scan status" />` (existing pattern).
- Icon-only collapsed sidebar CTA (below lg breakpoint): `aria-label="New Scan"` on the icon-only `<Button>`.
