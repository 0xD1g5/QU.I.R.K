## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

## Color

All colors are CSS variable references. No hardcoded `hsl()` literals anywhere in Phase 39 code (D-13 CSS variable audit — enforced by sidebar.tsx header comment and existing codebase convention).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` → `240 10% 4%` dark / `0 0% 100%` light | Page background |
| Secondary (30%) | `hsl(var(--card))` → `240 6% 10%` dark / `0 0% 98%` light | Cards, sidebar, table containers |
| Accent (10%) | `hsl(var(--accent))` → `210 100% 56%` (Sky 500) | Active nav item border + bg tint, row hover tint |
| Destructive | `hsl(var(--destructive))` → `0 72% 51%` dark / `0 72% 45%` light | CRITICAL severity badge only |

Accent reserved for: active sidebar nav item left-border + `bg-accent/10` background tint; `hover:bg-accent/5` table row hover state. Do NOT use accent for any other element in Phase 39.

Severity badge colors follow the `SEVERITY_STYLES` constant from `motion.tsx` — replicate verbatim:

| Severity | Background | Text |
|----------|-----------|------|
| CRITICAL | `bg-[hsl(0_72%_51%)]` | `text-white` |
| HIGH | `bg-[hsl(24_95%_53%)]` | `text-white` |
| MEDIUM | `bg-[hsl(38_92%_50%)]` | `text-black` |
| LOW | `bg-[hsl(213_94%_68%)]` | `text-black` |
| INFO | `bg-[hsl(240_5%_46%)]` | `text-white` |

Note: Severity badge colors are intentionally hardcoded `hsl()` values (not CSS variables) — this is the established project pattern from `motion.tsx`. Do not convert them.

Quantum-risk semantic tokens (reused from `ScoreGauge.tsx`):

| State | Token | Value |
|-------|-------|-------|
| Safe (≥ 80) | `hsl(var(--quantum-safe))` | Green 500 |
| At risk (50–79) | `hsl(var(--quantum-at-risk))` | Amber 500 |
| Vulnerable (< 50) | `hsl(var(--quantum-vulnerable))` | Red 600 |

Source: `src/dashboard/src/index.css`, `src/dashboard/src/pages/motion.tsx` (SEVERITY_STYLES), `src/dashboard/src/components/gauges/ScoreGauge.tsx`.

## Implementation Constraints

1. **No hardcoded `hsl()` outside SEVERITY_STYLES and boolean badge classes.** All other color references must use `hsl(var(--token))` CSS variable pattern. (D-13 audit, enforced by `sidebar.tsx` header comment.)
2. **`dar_findings` array on `useScanData()` response.** The hook passes through the full API response; once `ScanLatestResponse.dar_findings` is added to the backend schema, the frontend accesses it as `data?.dar_findings ?? []`.
3. **Route and nav must change in lockstep.** `App.tsx` Route block and `sidebar.tsx` NAV_ITEMS must both be updated in the same plan.
4. **ScoreGauge at size={120}.** Do not use `isOverall={true}` — that is reserved for the Executive tab's overall gauge.
5. **No drill-down panels.** Clicking a row does nothing in Phase 39. Phase 43 or a future phase will add drill-down evidence panels.
6. **Zero console errors gate.** The executor must verify the browser console is clean on the `/data-at-rest` route before marking the plan complete (CONTEXT.md phase boundary).
