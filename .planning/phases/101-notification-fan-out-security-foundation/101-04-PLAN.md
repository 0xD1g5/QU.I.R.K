<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| notification dispatch → scan record / scheduler loop | A delivery failure must never corrupt the committed scan record or stall the loop |
| exception text → integration_deliveries.error_summary / logs | Secrets in error text crossing into persisted audit rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-10 | Tampering / Denial of Service | _dispatch_schedule hook | mitigate | Hook runs AFTER the final db.commit() and is wrapped in try/except that swallows + safe_str-logs; scan record is already persisted (success criterion 3, NOTIFY-07) |
| T-101-11 | Information Disclosure | _deliver error_summary | mitigate | error_summary is always safe_str(exc); test injects an xoxb- token exception and asserts the stored row is scrubbed (ISEC-02) |
| T-101-12 | Denial of Service (alert fatigue) | should_notify | mitigate | Conservative trigger (new_high>0 OR score_delta<-floor); score_delta is None → no fire on first scan (Pitfall 4); never MEDIUM-only (NOTIFY-02) |
| T-101-13 | Denial of Service | per-channel fan-out | mitigate | Each channel isolated in _deliver try/except; one channel's failure does not block the others; channel timeouts enforced upstream in Plan 03 |
</threat_model>
