<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config file → NotifyCfg | Operator-supplied config; must yield env-var names only, never literal secrets |
| TrendReport (internal) → outbound payload dict | Crossing from internal infra-detail to third-party-bound JSON |
| scheduler --config (DB path) → notify config loader | Overloaded arg; a DB path fed to a YAML parser is a crash/leak risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-03 | Information Disclosure | to_integration_payload | mitigate | Single canonical whitelist excludes new_findings_sample host/port/protocol; test asserts absence even when source carries samples (ISEC-03) |
| T-101-04 | Information Disclosure | NotifyCfg fields | mitigate | Dataclasses store only `*_env` env-var NAMES; no literal-secret fields exist to persist (NOTIFY-06) |
| T-101-05 | Denial of Service | load_notifications_config on DB path | mitigate | try/except → None on any parse failure; never raises into the scheduler loop (Pitfall 1) |
</threat_model>
