  <name>Task 3: Write docs/release-process.md (version policy + runbook + attestation verification)</name>
  <files>docs/release-process.md</files>
  <action>
    Create `docs/release-process.md` with four major sections:

    1. `## Version Policy`
       - Semver commitments: MAJOR for breaking schema or CLI changes, MINOR for new scanners/features, PATCH for bugfixes.
       - EOL cadence: each MAJOR line receives security fixes for 12 months after the next MAJOR ships.
       - What triggers a bump: schema changes (MAJOR), new scanner / new public API surface (MINOR), bugfix / docs / chaos lab tweaks (PATCH).
       - Single source of truth: `pyproject.toml [project.version]` per D-84-R1; `quirk/__init__.py::__version__` derives via `importlib.metadata`; `tests/test_version.py` enforces parity across six surfaces.

    2. `## Release Runbook`
       Step-by-step ordered list:
       a. Verify all CI green on main.
       b. Bump `pyproject.toml [project.version]` to the new version (this is the only edit — every other surface derives from it).
       c. Run `towncrier build --version X.Y.Z --yes` — this consumes `changelog.d/*.md` fragments, prepends them to `CHANGELOG.md`, and removes the consumed fragments.
       d. Commit: `chore(release): vX.Y.Z` (explicit paths only: `git add pyproject.toml CHANGELOG.md changelog.d/`).
       e. Tag: `git tag vX.Y.Z` and push: `git push origin main --tags`.
       f. The push of the `vX.Y.Z` tag triggers `.github/workflows/release.yml` — monitor the GHA run.
       g. PyPI publish succeeds via Trusted Publishers (OIDC); Sigstore attestations are auto-generated.
       h. Verify the published release on PyPI carries the wheel + sdist + attestations.

    3. `## One-Time Setup (per project, never re-run)`
       - Configure PyPI Trusted Publisher: project name from `pyproject.toml`, repo owner/name, workflow filename `release.yml`, environment name `release`.
       - Create the GitHub `release` environment in repo Settings -> Environments. Optionally restrict to tag pattern `v*`.

    4. `## Attestation Verification (for downstream consumers)`
       - GitHub CLI: `gh attestation verify <path-to-wheel> --repo <owner>/<repo>` — returns 0 on success and prints the signing identity.
       - cosign alternative: `cosign verify-blob --certificate-identity ...` — full command with the OIDC issuer URL for GitHub Actions.
       - Note: `curl | bash` installers are explicitly NOT shipped — this is a deliberate non-feature per LAUNCH-07; downstream consumers must use PyPI / Homebrew / GHCR which all carry signed provenance.
  </action>
