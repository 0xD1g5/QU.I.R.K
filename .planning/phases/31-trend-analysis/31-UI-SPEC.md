**Accent reserved for:**
- Active nav indicator (border-left + `bg-accent/10` background) on the Trends sidebar item

Score delta badge is color-coded by sign:
- Positive delta ŌåÆ `bg-[hsl(var(--quantum-safe))] text-white` with `Ō¢▓ +X.X pts` prefix

- Zero delta ŌåÆ `bg-[hsl(var(--muted))] text-muted-foreground` with `ŌåÆ 0.0 pts`

## Page Layout

The `/trends` page follows the exact established pattern from `findings.tsx` and `sidebar.tsx`:

```
<div className="space-y-4">
  <h1>Trends</h1>              ŌåÉ page heading, fontSize: 20, fontWeight: 600
  <header row>                 ŌåÉ "Comparing [prev_ts] ŌåÆ [current_ts]" or empty state
  <cards row (3 cards)>        ŌåÉ Score delta | New findings | Resolved findings
  <scan error row>             ŌåÉ compact single-line below cards
  <collapsible table>          ŌåÉ New Findings top-5 (hidden when count = 0)
  <collapsible table>          ŌåÉ Resolved Findings top-5 (hidden when count = 0)
</div>
```

**Card layout:** 3 cards in a responsive grid ŌĆö `grid grid-cols-1 md:grid-cols-3 gap-4`

**Score delta card internal layout:**
```
CardTitle: "Readiness Score"
CardContent:
  Previous score (muted) ŌåÆ Current score (foreground, 28px semibold) ŌåÆ delta badge
```

**New / Resolved finding cards internal layout:**
```
CardTitle: "New Findings" / "Resolved Findings"
CardContent:
  HIGH: [count]   (severity-high color)
  MEDIUM: [count] (severity-medium color)
  LOW: [count]    (severity-low color)
```

**Scan error row:** Single `<p>` below the card grid:
```
<p className="text-xs text-muted-foreground">
  Scan coverage: +{scan_errors_new_count} new errors, -{scan_errors_resolved_count} resolved errors
</p>
```
Omit entirely if both counts are 0.

**Session timestamp format:** `toLocaleString('en-US', { month: 'short', day: 'numeric', hour: '2-digit', minute: '2-digit' })` ŌĆö e.g. "Apr 26, 09:00 AM". Applied to both `current_session_ts` and `previous_session_ts`.

## Hook Contract

New `useTrendsData` hook in `src/dashboard/src/hooks/useTrendsData.ts`:

```ts
// Pattern: mirrors useScanData exactly
// Returns: { data: TrendReport | null, loading: boolean, error: string | null }
// Fetches: GET /api/trends
// No parameters ŌĆö always fetches the two most recent sessions
```

TypeScript type for API response (mirrors D-07 exactly):

```ts
interface TrendReport {
  current_session_ts: string           // ISO datetime
  previous_session_ts: string | null   // null when only 1 session exists
  current_score: number | null
  previous_score: number | null
  score_delta: number | null           // null when no previous session
  new_high: number
  new_medium: number
  new_low: number
  resolved_high: number
  resolved_medium: number
  resolved_low: number
  scan_errors_new_count: number
  scan_errors_resolved_count: number
  new_findings_sample: SampleFinding[]
  resolved_findings_sample: SampleFinding[]
}

interface SampleFinding {
  host: string
  port: number
  protocol: string
  severity: string
}
```
