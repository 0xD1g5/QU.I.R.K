K8S-01 detection ladders Plan 02 will implement (these tests pin them):

EKS encryptionConfig:
| describe_cluster response                                       | service_detail              | severity |
|-----------------------------------------------------------------|-----------------------------|----------|
| encryptionConfig absent OR empty list                            | "EKS/unencrypted"           | "HIGH"   |
| encryptionConfig with resources=["secrets"] + provider.keyArn    | "EKS/encrypted:{keyArn}"    | None     |

GKE database_encryption.current_state (int enum):
| current_state value                                              | service_detail              | severity |
|------------------------------------------------------------------|-----------------------------|----------|
| 2 (CURRENT_STATE_ENCRYPTED)                                      | "GKE/encrypted:{key_name}"  | None     |
| 0 / 1 / other (UNSPECIFIED / DECRYPTED)                          | "GKE/unencrypted"           | "HIGH"   |

AKS security_profile.azure_key_vault_kms.enabled:
| nested getattr result                                            | service_detail              | severity |
|------------------------------------------------------------------|-----------------------------|----------|
| enabled == True                                                  | "AKS/kv-kms"                | None     |
| enabled == False / security_profile None / kv_kms None           | "AKS/platform-managed"      | "MEDIUM" |

- list_namespaced_secret().items returns V1Secret objects; access only secret.type
- Counter(s.type or "Opaque" for s in items) produces type counts

- ApiException with status==403 → CryptoEndpoint with scan_error="insufficient-rbac-privileges"

K8S-03 mandatory paths (never silent empty):
- provider not in {"eks","gke","aks"} → 1 endpoint, scan_error="encryption-config-inaccessible"
- K8S_AVAILABLE=False AND provider valid → 1 endpoint, scan_error="encryption-config-inaccessible"
