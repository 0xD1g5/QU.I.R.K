to output JSON and Plan 05 surfaces on the dashboard.

partial_failures accumulation. The existing bulk DB persist at line ~1119 is
retained for the endpoint types not covered by per-stage flush (error_endpoints
themselves, broker metadata); it becomes a no-op for already-persisted rows
because SQLAlchemy tracks identity.
</objective>

    ```python
    def _flush_stage_endpoints(db_path: str, endpoints: list) -> None:
        """Phase 67 RESUME-01: flush a stage's CryptoEndpoint rows to SQLite immediately.

        Called after each scanner stage so a crash between stages leaves results
        for completed stages persisted. Silent no-op on failure — the scan's
        bulk persist at the end is the safety net.
        """
        if not endpoints or not db_path:
            return
        try:
            with get_session(db_path) as session:
                for ep in endpoints:
                    session.merge(ep)   # merge: safe if row already exists
                session.commit()
        except Exception:
            pass
    ```

    Use `session.merge(ep)` rather than `session.add(ep)` to handle the case
    where a resume re-runs a stage and attempts to persist an endpoint whose
    primary key already exists. `merge()` is idempotent for existing rows.

    ```python
    def _collect_stage_partial_failures(
        run_stats: dict,
        stage: str,
        error_endpoints: list,
        previous_error_count: int,
    ) -> list:
        """Phase 67 RESUME-02: build partial_failures entries from new error_endpoints.

        Compares error_endpoints length before vs after a stage to find new errors.
        Returns the list of new partial_failure dicts (also appended to run_stats).
        """
        new_errors = error_endpoints[previous_error_count:]
        stage_failures = []
        for ep in new_errors:
            stage_failures.append({
                "stage": stage,
                "scanner": getattr(ep, "host", "unknown"),
                "error_category": getattr(ep, "scan_error_category", "exception"),
                "error_message": getattr(ep, "scan_error", "") or "",
                "endpoint_count": 0,
            })
        run_stats.setdefault("partial_failures", []).extend(stage_failures)
        return stage_failures
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scan subprocess → SQLite | Stage endpoints flushed incrementally; same trust level as existing bulk persist |
| error_endpoints list → partial_failures dict | Scanner exception messages serialized into run_stats; no external input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-02-01 | Tampering | _flush_stage_endpoints session.merge | accept | merge() is used intentionally for idempotency on resume; no external actor can influence which rows are merged (scan process only) |
| T-67-02-02 | Information Disclosure | partial_failures in run_stats | accept | run_stats written to quirk-output/ which has same access controls as other scan output; scanner exception messages contain no user PII |
| T-67-02-03 | Denial of Service | incremental flush adding DB write overhead | accept | Each flush is a small bounded set of rows per stage; the performance impact is negligible compared to scanner I/O |
</threat_model>
