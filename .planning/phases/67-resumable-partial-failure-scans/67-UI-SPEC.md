## UI Surfaces in Scope

This phase adds exactly two UI surfaces:

**Surface 1 — Dashboard "Scanner Status" card (executive.tsx)**
A conditional Card component inserted into the Executive Summary page. Rendered only when `partial_failures` from `GET /api/scan/latest` is non-empty. Hidden for clean scans — no empty state needed.

**Surface 2 — CLI `quirk scan --list-resumable` (Python/rich)**
Tabular terminal output via `rich.table.Table`. Not a browser UI. Design contract for this surface covers column layout, color tokens, and copy only — no React or Tailwind applies.

## Card Layout Specification

**Placement:** Inside the `<div className="space-y-8">` root of ExecutivePage, after the Severity Breakdown card and before the scan metadata `<p>` line.

**Render condition:** `{data.partial_failures && data.partial_failures.length > 0 && ( <ScannerStatusCard failures={data.partial_failures} /> )}`

**Card internal layout (per row):**

```
[Stage label — muted 14px]  [Scanner label — 14px regular]  [Badge]  [Error category — muted 14px]  [Error message — 14px regular, truncated at 60ch]
```

Row layout: `flex items-center gap-3 py-2 border-b border-border last:border-0`

**Card header:** Title "Scanner Status" at 20px/600 weight. No subtitle. No action buttons.

**Collapsible:** Not required for this phase. The card is always expanded when visible. If partial_failures exceeds 8 rows in practice, a future phase can add collapsible. For Phase 67: always show all rows.

## Interaction Contract

**Scanner Status card:**
- Read-only. No click targets, no hover states on rows.
- Card is always visible when rendered (no toggle). Not collapsible in Phase 67.
- Data fetched by the existing `useScanData` hook (which already calls `GET /api/scan/latest`). No new hook needed — the hook result is extended with `partial_failures[]`.
- Hook cancellation: if a new hook is needed for any reason, it MUST follow the `let cancelled = false` / `return () => { cancelled = true }` pattern from Phase 62.
- Loading state: inherited from existing `useScanData` loading/error handling in ExecutivePage. Scanner Status card is only rendered after data is available.

**CLI --list-resumable:**
- One-shot command output. No interactivity.
- Table rendered via `rich.table.Table` with `show_header=True`, `header_style="bold"`.
- Age column formatted as `{N}h {M}m` for sub-24h, `{N}d {M}h` for multi-day.
- Target column shows the scan target if recoverable from `scan_jobs` join; otherwise displays `—` (em dash).
- Age > 72h: row highlighted with `style="yellow"` to draw attention without blocking.

## Accessibility

- Badge elements must include `aria-label` when status meaning is color-only. Use `aria-label="status: Partial"` / `aria-label="status: Failed"` / `aria-label="status: Skipped"`.
- Card title rendered as `<CardTitle>` (maps to h3 semantically via shadcn defaults).
- Error message column: if truncated with CSS `truncate`, add `title={full_error_message}` on the cell element for tooltip on hover.
- Color is never the sole differentiator — badge text label always present alongside color.
