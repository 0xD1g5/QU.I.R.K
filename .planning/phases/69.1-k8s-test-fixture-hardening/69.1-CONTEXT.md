# Phase 69.1: K8s Test Fixture Hardening - Context

**Gathered:** 2026-05-15
**Status:** Ready for planning
**Parent:** Phase 69 (Deferred BLOCKERs — Scanner + Cloud)

<domain>
## Phase Boundary

Phase 69.1 closes the test-infrastructure gap discovered during Phase 69's verification: 6 `tests/test_k8s_connector.py` tests fail under the project's canonical `.venv` (Python 3.14) because they patch `AKS_AVAILABLE` / `GKE_AVAILABLE` but not `K8S_AVAILABLE`. The `kubernetes` Python SDK is not installed in `.venv`, so the flag defaults to `False` and `scan_k8s_targets` short-circuits via K8S-03 path B (line 405) before any test-specific patch is exercised.

**In scope:** Single test-file hardening — add `K8S_AVAILABLE` patching (and any companion `*_AVAILABLE` flags) to all affected tests so they run cleanly under `.venv` regardless of which optional cloud-SDK packages are installed.

**Out of scope:**
- Installing `kubernetes`/`azure-mgmt-containerservice`/`google-cloud-container` into `.venv` (those are opt-in cloud extras per `pyproject.toml`; tests must not require them).
- Production source code changes (the Phase 69-03 fix at `k8s_connector.py:502` is correct — verified independently by gsd-verifier; no change here).
- Other test files (only `test_k8s_connector.py` exhibits this pattern).

</domain>

<decisions>
## Implementation Decisions

### D-01: Conftest auto-fixture (preferred)

Add a module-level autouse fixture in `tests/test_k8s_connector.py` (or a co-located `tests/conftest.py` scoped to this file) that patches all three `*_AVAILABLE` flags to `True` for every test in the file:

```python
@pytest.fixture(autouse=True)
def _k8s_available_flags():
    with patch("quirk.scanner.k8s_connector.K8S_AVAILABLE", True), \
         patch("quirk.scanner.k8s_connector.AKS_AVAILABLE", True), \
         patch("quirk.scanner.k8s_connector.GKE_AVAILABLE", True):
        yield
```

Rationale: tests in this file all exercise `scan_k8s_targets` and rely on the flags being True to reach the SDK-dispatch branches. Patching at fixture level keeps individual tests focused on their specific assertions instead of repeating the flag dance. Tests that *want* to exercise the K8S-03 path-A/B fallbacks (provider unsupported, K8S SDK absent) can override the fixture inline via `patch("...K8S_AVAILABLE", False)`.

### D-02: No production source code changes

The Phase 69 source fixes at `k8s_connector.py:502` (D-09 short-circuit), `:454-510` (credential-failure inaccessible-finding paths), and surrounding logic are all verified correct by gsd-verifier. This phase only touches the test file.

### D-03: Verification

All 5 currently-failing tests must pass after the fix:
- `test_aks_empty_cluster_list_returns_empty` (Phase 69-03 test)
- `test_aks_credential_failure_emits_inaccessible_per_cluster` (pre-existing)
- `test_secret_rbac_403_produces_insufficient_privileges` (pre-existing)
- `test_gke_encrypted_no_severity` (pre-existing)
- `test_gke_unencrypted_produces_high` (pre-existing)

Plus regression: no test that *was* passing should now fail. Full `pytest tests/test_k8s_connector.py -q` must report ≥ 18 passed (current: 13 passed + 5 failed = 18 total; after fix: 18 passed + 0 failed).

</decisions>

## Existing Code Insights

`quirk/scanner/k8s_connector.py` defines three module-level availability flags set at import time:

```python
try:
    import kubernetes  # noqa: F401
    K8S_AVAILABLE = True
except ImportError:
    K8S_AVAILABLE = False

try:
    from azure.mgmt.containerservice import ContainerServiceClient as _AKSClient  # noqa: F401
    AKS_AVAILABLE = True
except ImportError:
    AKS_AVAILABLE = False

try:
    from google.cloud import container_v1 as _gke_container  # noqa: F401
    GKE_AVAILABLE = True
except ImportError:
    GKE_AVAILABLE = False
```

In the canonical `.venv`, all three are `False` (cloud SDKs are opt-in extras). The conftest fixture patches them at runtime for tests that exercise the SDK-dispatch branches.

</code_context>

<specifics>
## Specific Implementation

Add to `tests/test_k8s_connector.py` (near the top, after the imports):

```python
import pytest
from unittest.mock import patch

@pytest.fixture(autouse=True)
def _k8s_available_flags():
    """Phase 69.1: patch *_AVAILABLE flags so tests work in .venv without optional cloud SDKs.

    Tests in this file exercise scan_k8s_targets's SDK-dispatch branches. The
    module-level flags default to False in .venv because the cloud SDKs are
    opt-in extras. Tests that want to exercise the K8S-03 path A/B fallbacks
    can override via patch("...K8S_AVAILABLE", False) within the test body.
    """
    with patch("quirk.scanner.k8s_connector.K8S_AVAILABLE", True), \
         patch("quirk.scanner.k8s_connector.AKS_AVAILABLE", True), \
         patch("quirk.scanner.k8s_connector.GKE_AVAILABLE", True):
        yield
```

Then remove the now-redundant per-test `with patch("...AKS_AVAILABLE", True):` / `GKE_AVAILABLE` blocks throughout the file (cleanup; not strictly required since outer patches no-op when value matches).

</specifics>
