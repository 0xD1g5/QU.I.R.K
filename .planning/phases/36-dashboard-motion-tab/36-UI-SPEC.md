### Email Table columns (per DASH-02, CONTEXT.md D-02)

Ordered: Port | Protocol | TLS Version | Cipher Suite | Cert Expiry | Quantum Risk | Warning

- **Port** ŌĆö numeric, `text-sm`
- **Protocol** ŌĆö scanner-emitted label verbatim (e.g., `SMTP-STARTTLS`, `SMTPS`)
- **TLS Version** ŌĆö `tls_version` field, `text-sm font-mono`; empty cell if `plaintext_exposed`
- **Cipher Suite** ŌĆö `cipher_suite` field, `text-sm font-mono`; empty cell if `plaintext_exposed`
- **Cert Expiry** ŌĆö `cert_not_after` ISO date, formatted `toLocaleDateString("en-US", { dateStyle: "medium" })`; empty cell if absent
- **Quantum Risk** ŌĆö `quantum_risk` text, `text-sm`
- **Warning** ŌĆö render `ŌÜĀ STARTTLS` badge only when `starttls_warning === true`; empty cell otherwise

### Broker Grouped Sections (per DASH-03, CONTEXT.md D-03)

Three subsections in fixed display order: **Kafka**, **AMQP**, **Redis**.

Each subsection:
```
<Card>
  <CardHeader className="pb-2">
    <CardTitle className="text-sm font-semibold flex items-center gap-2">
      {family} ┬Ę {count} endpoint(s) ┬Ę {plaintextCount} plaintext
```

Always-expanded (no accordion). Subsection hidden entirely if no findings for that family in the current scan (family with zero endpoints is not rendered ŌĆö this is distinct from the "no broker endpoints at all" empty-state card which replaces the entire broker section).

**Broker table row columns:** Host | Port | Protocol | TLS Version | Cipher Suite | Status

- **Protocol** ŌĆö verbatim scanner label
- **Status cell** ŌĆö render `ŌśĀ PLAINTEXT` badge when `plaintext_exposed === true`; render `Ōśü {suffix}` cloud chip alongside if `protocol.includes('/')` (suffix = `protocol.split('/')[1]`); no badge if healthy TLS row

## Family Detection Logic

Executor must implement exactly as specified in CONTEXT.md D-03. Included here for renderer clarity:

```ts
function getBrokerFamily(protocol: string): "Kafka" | "AMQP" | "Redis" | null {
  if (protocol.startsWith("KAFKA-")) return "Kafka"
  if (protocol.startsWith("AMQP-") || protocol.startsWith("AMQPS")) return "AMQP"
  if (protocol.startsWith("REDIS-")) return "Redis"
  return null  // unknown protocol ŌĆö skip silently
}

function isEmailProtocol(protocol?: string): boolean {
  return ["SMTP-STARTTLS", "SMTPS", "IMAP-STARTTLS", "IMAPS", "POP3-STARTTLS", "POP3S"]
    .includes(protocol ?? "")
}
```
