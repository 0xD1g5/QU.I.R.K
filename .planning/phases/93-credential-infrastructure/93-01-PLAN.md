## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → CLI/env/@file | Untrusted secret reference enters the process; raw secret must never reach argv |
| in-memory secret → str materialization | bytearray→str happens only at as_headers()/query_param() (httpx boundary) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-93-01 | Information Disclosure | CLI argv leak of raw secret (ps/history) | mitigate | D-01: flags carry references only (@file / env name / bare→getpass); inline-secret misuse raises ValueError with guidance |
| T-93-02 | Information Disclosure | Secret residency in process heap | mitigate (best-effort) | D-04: bytearray storage + close() zeroization; D-05 best-effort caveat documented in Plan 03 gate |
| T-93-03 | Information Disclosure | Env var inherited by subprocess forks | mitigate | from_cli deletes consumed env var (PITFALLS Pitfall 1 step 5) |
| T-93-04 | Tampering | @file path traversal (read /etc, /proc) | mitigate | Reuse v4.8 load_targets_file path-traversal + blocked-prefix guard; TargetFileError propagates |
| T-93-05 | Information Disclosure | Secret in CredentialContext repr/logs/ValueError | mitigate | _secret_buf field repr=False; from_cli error text routed through safe_str; test asserts sentinel absent from repr |
| T-93-SC | Tampering | npm/pip installs | accept | No new dependencies introduced (stdlib getpass/os/re only — D per STACK.md) |
</threat_model>
