Algorithm name normalization (RESEARCH.md §Algorithm Name Normalization lines 466-end):
- Build a `_FAMILY_MAP: dict[str, str]` from lowercased mode-specific names
  (aes-128-gcm, aes-256-gcm, ecdh-sha2-nistp256, rsa-sha2-256, etc.) to CAVP family
  names (AES, ECDSA, RSA, SHS, HMAC, DRBG, KAS, ...).
- `normalize_for_cmvp_lookup(name: str) -> str | None`:
  return _FAMILY_MAP.get(name.lower(), None); fallback regex maps `aes-*` → "AES",
  `rsa*` → "RSA", `ec(dh|dsa)*` → "ECDSA", `sha-*|sha[0-9]+` → "SHS",
  `hmac-*` → "HMAC". Returns None for non-CMVP algos (e.g. ChaCha20) so the
  caller renders "Not in CMVP catalog".
