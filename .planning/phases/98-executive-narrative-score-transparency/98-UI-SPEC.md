# Phase 98 — UI Design Contract

> Visual and interaction contract for Phase 98: Executive Narrative + Score Transparency.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Clarification

Phase 98 is primarily a **backend content** phase. The UI surface delta is narrow:

1. **HTML report template** (`report.html.j2`) — add narrative prose block, top-risks section, rollup formula explanation, and impact×effort priority ordering for roadmap items. No new components; pure HTML/CSS additions within the existing embedded stylesheet.
2. **CLI markdown** (`executive.py`) — already outputs Score Decomposition. Narrative, top-risks, and roadmap ordering come from the shared content model; no new terminal UI primitives.
3. **React dashboard** (`executive.tsx`) — NO layout changes. The six ScoreGauge subscore row already exists. Phase 98 does not add any new dashboard components or interactive elements.
4. **PDF** — derived from HTML via Playwright; no separate PDF CSS work (Phase 100 owns PDF branding).

All design decisions below apply to the HTML report template surface unless otherwise noted.

- Narrative prose block: `--text` (body copy), no new role

## Interaction Contract

### HTML Report (static document — no JS interactions)

All Phase 98 additions are static HTML/CSS. No JavaScript, no interactive elements added to the report template.

**Narrative prose block** — Positioned immediately after the `<h1>Executive Summary</h1>` and before the score card. Static `<p>` elements. No interactivity.

**Top-risks section** — Positioned after the score decomposition table and before "Findings Breakdown". Rendered as an unordered list using the existing `.drivers-list` class pattern (or a new `.risks-list` variant with per-item severity color coding). No interactivity.

**Roadmap items with priority** — Extend existing `.roadmap-item` `<div>` with a `<span class="priority-label">` showing effort/impact. No interactivity.

**Score rollup formula** — A `<p>` block below the decomposition table, visible on every report. Static.

### React Dashboard (no interaction changes)

No new interactive elements. The six ScoreGauge row is already implemented and unchanged.
