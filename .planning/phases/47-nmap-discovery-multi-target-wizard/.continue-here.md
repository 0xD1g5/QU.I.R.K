## Critical Anti-Patterns (do NOT repeat these)

- **ANTI-PATTERN: file-overlap parallelism** — two plans extending the same dataclass + registry in different waves is a merge-conflict trap. Structural mitigation: linearize via `depends_on` (NOT "merge it later").
- **ANTI-PATTERN: locked-literal as helper kwarg default** — when CONTEXT says "X is not configurable," do not park X as a kwarg default "for testability." Pass it explicitly at the call site with an inline `# D-NN locked` comment so the invariant survives future edits.
- **ANTI-PATTERN: narrow try/except** — wrapping only the immediate call (`validate_str`) misses constructor-time exceptions from the same library. When a missing-deps exception can fire at multiple sites, wrap them all in one try-block.
- **ANTI-PATTERN: `.num_addresses` for probe-budget math on /24** — off-by-2 on IPv4 (excludes network + broadcast). Use `.hosts()` length OR formula `num_addresses - (2 if v4 and prefixlen<31 else 0)` (RESEARCH risk #4).
