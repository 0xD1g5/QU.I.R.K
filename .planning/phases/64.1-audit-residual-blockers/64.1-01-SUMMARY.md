# Phase 64.1 Plan 01: Audit Residual Blockers — 5 Bug Fixes + Ledger Closures Summary

**One-liner:** Five targeted audit bug fixes (des→DES mapping, date arithmetic staleness, years input clamp, microsecond session precision, init_db idempotency) with a regression test for each, closing 5 ledger BLOCKERs atomically.

## What Was Built

### Task 1: CR-03 — Fix algo hints in _extract_algo_from_rule_id

Fixed the ordered `algo_hints` list in `quirk/cbom/builder.py` with three changes:
1. `("des", "3DES")` → `("des", "DES")` — DES and 3DES are distinct algorithms
2. Added `("aes-256", "AES-256")` and `("aes-128", "AES-128")` before the generic `("aes", ...)` entry (ordered-scan invariant: more-specific before less-specific)
3. `("aes", "AES-256-GCM")` → `("aes", "AES")` — no variant assumption from semgrep rule ID alone

Created `tests/test_cbom_builder_algo_hints.py` with 8 parametrized cases covering all 6 spec scenarios plus explicit des_not_3des and 3des_still_works tests.

Lab expected_results check: `grep -r "3DES\|AES-256-GCM" quantum-chaos-enterprise-lab/expected_results_v3.md quantum-chaos-enterprise-lab/expected_results_v4.md` — no matches. The chaos lab profiles do not exercise `_extract_algo_from_rule_id` in a way that produces the old wrong values; no lab file updates required.

**Commit:** b54230d

### Task 2: BL-03 — Fix staleness date comparison in status_report

Added `import datetime` to `quirk/compliance/__init__.py` (was absent). Replaced the bare string comparison:
```python
if key not in seen or e["last_verified"] < seen[key]["last_verified"]:
```
with:
```python
if (
    key not in seen
    or datetime.date.fromisoformat(e["last_verified"])
    < datetime.date.fromisoformat(seen[key]["last_verified"])
):
```
This matches the canonical pattern in `quirk/cli/doctor_cmd.py:51-55`.

Created `tests/test_compliance_status_staleness.py` with 3 tests: oldest-kept behavioral, date-arithmetic structural (inspects source for `datetime.date.fromisoformat`), and malformed-date ValueError.

**Commit:** 4d2c4e2

### Task 3: BL-04 — Years input clamp at both interactive entry points

Added `if years < 1: raise ValueError(...)` guard after the `try/except` block in both:
- `quirk/interactive.py` (Data longevity years [7] prompt)
- `quirk/assessment/operator_context.py` (years confidential prompt)

Non-numeric input still falls through to default-7 via the existing `except` path. Explicit zero or negative values raise `ValueError("Data longevity years must be at least 1 (got: {years_raw!r})")`.

Created `tests/test_operator_context_years_clamp.py` with 14 tests: 2 source-structure assertions + 6 behavioral cases × 2 parsing-logic fixtures (interactive and operator_context blocks).

**Commit:** 73dfac1

### Task 4: CR-05 — Microsecond strftime precision for session grouping

Changed strftime format from `%Y-%m-%d %H:%M:%S` to `%Y-%m-%d %H:%M:%f` at two call sites in `quirk/dashboard/api/routes/trends.py` (`_list_session_timestamps` and `_list_session_timestamps_n`). Renamed the label from `ts_sec` to `ts_usec` and updated module/function docstrings.

In `quirk/intelligence/trends.py`, changed `_fetch_session_endpoints` window from `timedelta(seconds=1)` to `timedelta(microseconds=1)` and updated the docstring from "±1 second window" to "1-microsecond window (one canonical scanned_at timestamp)".

Created `tests/test_trends_subsecond_sessions.py` with 4 tests: two-endpoint disambiguation, microsecond round-trip precision, fetch-only-exact-ts isolation, and NULL exclusion (D-13 regression).

All 23 existing trend tests still pass.

**Commit:** 01d8a6d

### Task 5: CR-08 — init_db idempotency regression test

Confirmed the 9 engine-accepting `_ensure_*` functions in `quirk/db.py` are already idempotent (each guards with `if col not in existing` via `sa_inspect`). No source change to `quirk/db.py` needed.

- `test_init_db_twice_on_fresh_db`: schema unchanged after double init

- `test_init_db_after_simulated_partial_migration`: base-only schema catches up to full schema via `init_db()`
