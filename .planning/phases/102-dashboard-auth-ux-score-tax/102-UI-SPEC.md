# Phase 102 — UI Design Contract

> Visual and interaction contract for the AUTH-03 login surface only.
> Non-UI deliverables (AUTH-01 token CLI, AUTH-02 X-API-Key backend, TRANS-04 score refactor) are OUT OF SCOPE for this contract.

## Scope

This contract covers exactly four surfaces introduced by AUTH-03:

1. **Login page** — full-screen unauthenticated state; token input + submit button
2. **Inline error state** — invalid token feedback, rendered inside the login card
3. **Authenticated-state affordance** — logout control shown in the existing Sidebar once the user is authenticated
4. **Auth-init loading state** — brief probe-request in flight before the 401/200 decision resolves

Everything else (sidebar navigation, page content, existing charts, CBOM, etc.) is NOT modified by this phase and is NOT governed by this contract.

## Layout Contract

### Login Page (unauthenticated state)

- Replaces the entire `<main>` + `<Sidebar>` tree — the sidebar is NOT rendered when the user is unauthenticated
- Outer container: `flex min-h-screen items-center justify-center bg-background` — same `bg-background` (#080a0f) as the authenticated shell
- Login card: `Card` primitive, fixed width `w-full max-w-sm` (384px), centered, with `shadow-lg`
- Card structure:
  - `CardHeader`: product wordmark + page heading
  - `CardContent`: token input field + submit button + inline error slot
  - `CardFooter` (optional): muted helper text (see Copywriting)
- No sidebar, no nav, no top bar in this state

### Authenticated-State Logout Control

- Lives at the bottom of the existing `Sidebar` component, below the current `ModeToggle` / `ScanSelector` slot
- Collapsed sidebar (w-12, mobile/tablet): icon-only `Button` with `variant="ghost"` and `size="icon"`, tooltip "Sign out"
- Expanded sidebar (w-60, desktop): `Button` with `variant="ghost"` full-width, icon + label "Sign out"
- Icon: `LogOut` from lucide-react (already imported pattern — no new dep)
- Placement: `mt-auto` pushes it to the very bottom of the flex column, separated from nav items by a `Separator` (already in ui/)

### Auth-Init Loading State

- App boots → shows the authenticated shell skeleton for ≤ one render frame, then resolves
- Preferred: keep the loading state invisible (zero flash) by not mounting the sidebar/routes tree until the probe result is known
- Implementation note: AuthContext initialises as `status: "loading"` → render `null` or a minimal full-screen `bg-background` div (no spinner required — the probe is a single fast health fetch)

### LoginPage component (`src/dashboard/src/pages/login.tsx` — new file)

```
<div class="flex min-h-screen items-center justify-center bg-background">
  <Card class="w-full max-w-sm shadow-lg">
    <CardHeader>
      <span class="text-accent font-black text-base tracking-widest font-mono">QU.I.R.K.</span>
      <CardTitle class="text-xl font-semibold mt-2">Dashboard Login</CardTitle>
      <CardDescription>Enter your dashboard token to continue.</CardDescription>
    </CardHeader>
    <CardContent class="space-y-4">
      <div class="space-y-2">
        <Label htmlFor="token-input">API Token</Label>
        <Input
          id="token-input"
          type="password"
          placeholder="Paste your token"
          autoComplete="current-password"
        />
      </div>
      <!-- Error slot: conditionally rendered, never layout-shifting -->
      <p class="text-sm text-destructive" role="alert" aria-live="polite">
        Invalid token. Check your token and try again.
      </p>
      <Button type="submit" class="w-full">Unlock Dashboard</Button>
    </CardContent>
    <CardFooter>
      <p class="text-xs text-muted-foreground">
        Generate a token with: <code class="font-mono">quirk token generate</code>
      </p>
    </CardFooter>
  </Card>
</div>
```

Key constraints:
- Input `type="password"` — prevents token from appearing in plaintext; browser password manager eligible
- Error `<p>` is always in the DOM (not conditionally mounted) to avoid layout shift; controlled by `aria-hidden` + empty string when no error
- Submit triggers the probe + localStorage write; no page navigation occurs — the React auth state transitions render the authenticated shell

### AuthContext (`src/dashboard/src/context/AuthProvider.tsx` — new file)

State shape:
```
type AuthStatus = "loading" | "authenticated" | "unauthenticated"
interface AuthState {
  status: AuthStatus
  setToken: (token: string) => void   // called by LoginPage on success
  logout: () => void                   // clears localStorage, sets unauthenticated
}
```

Mounting guard in `App.tsx`:
- `status === "loading"` → render `<div class="min-h-screen bg-background" />` (blank dark screen, no flash)
- `status === "unauthenticated"` → render `<LoginPage />` (no sidebar, no routes)
- `status === "authenticated"` → render existing sidebar + routes tree unchanged

### Logout Control (added to `src/dashboard/src/components/sidebar.tsx`)

Placed at the bottom of the Sidebar `<aside>`, after existing nav and `ModeToggle`:

## Interaction Contract

### Auth flow (state machine)

```
App boots
  └─> AuthContext probe fetch (GET /api/health or first data route)
        ├─> 200 OK → status = "authenticated" → render dashboard
        └─> 401 Unauthorized → status = "unauthenticated" → render LoginPage

LoginPage: user enters token + submits
  └─> Probe fetch with token as X-API-Key header
        ├─> 200 OK → localStorage.setItem("quirk_api_token", token) → status = "authenticated"
        └─> 401 → show inline error, clear input value, keep focus on input

Logout (sidebar control)
  └─> localStorage.removeItem("quirk_api_token") → status = "unauthenticated" → render LoginPage
```

### Token injection

- On every API fetch: read `localStorage.getItem("quirk_api_token")`; if present, add header `X-API-Key: <token>`
- Injected in the existing fetch/client layer (`src/dashboard/src/lib/`) — a single intercept point, not scattered per-hook
- If a fetch returns 401 while authenticated (e.g. token rotated by another session): treat as logout — clear storage, set status = "unauthenticated"

### Input behaviour

- Token input is `type="password"` — masked by default
- No "show/hide" toggle required (out of scope; single-tenant use case)
- `autoComplete="current-password"` — allows browser/password manager autofill
- Submit on Enter key (native form submit) — wrap LoginPage content in `<form onSubmit={...}>`
- Button disabled + shows no spinner during probe — the probe is fast; no loading indicator needed
- After a failed attempt: input value cleared, focus returned to input, error message visible

### Focus management

- On LoginPage mount: auto-focus the token input (`autoFocus` prop on Input)
- After failed login: `inputRef.current?.focus()` — re-focus for rapid retry
- Tab order: Input → Submit button → (footer link if any)

### Auth-disabled passthrough

- If `quirk_api_token` is absent in localStorage AND the probe returns 200 (auth disabled server-side): status = "authenticated" immediately
- LoginPage is never shown when auth is disabled — the UX is identical to the pre-Phase-102 state

## Accessibility Contract

- Login `<form>` has `aria-label="Dashboard login"` on the form element
- Input has `id="token-input"` paired with `<Label htmlFor="token-input">`
- Error paragraph has `role="alert"` and `aria-live="polite"` — screen reader announces on failure without interrupting typing
- Submit button has a visible text label ("Unlock Dashboard") — no icon-only submit
- Logout icon button in collapsed sidebar has `aria-label="Sign out"` (icon only, no visible text)
- Focus ring uses `--ring` (teal, #4ba8a8) — visible against `--card` (#0d0f14) background; contrast is sufficient on the dark theme
