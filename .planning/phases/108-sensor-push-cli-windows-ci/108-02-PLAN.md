<objective>
Build the core sensor-side runtime: quirk sensor enroll (SENSOR-01), quirk sensor push (SENSOR-02), and store-and-forward spool (SENSOR-03), all in quirk/cli/sensor_cmd.py, wired into run_scan.py via the established lazy-dispatch pattern.

Purpose: This is the sensor agent. enroll binds the sensor to a console and mints the one-time token + persistent push credentials in sensor.yaml. push runs a local scan unchanged via subprocess, serializes endpoints into the byte-identical wire envelope, signs it with HMAC, and POSTs over verified HTTPS with tenacity retry — spooling to a bounded dir when offline. The wire-envelope builder and compression helper defined here are the single source consumed by Plan 03 (export/import) so the export payload is byte-identical to the push body (the most important Phase 109/110 invariant).

Output: quirk/cli/sensor_cmd.py, run_scan.py sensor dispatch block, two test files.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sensor process → console over HTTPS | untrusted network; TLS + HMAC + Bearer auth cross here |
| operator CLI input → sensor.yaml | console_url and api-token are operator-supplied |
| sensor.yaml on disk | holds hmac_key + console_api_token (push credentials) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-108-03 | Tampering | TLS downgrade via verify=False | mitigate | httpx.Client verify=True hardcoded, no override param; tests/test_sensor_no_verify_false.py comment-stripped grep gate over sensor_cmd.py + console_cmd.py |
| T-108-04 | Spoofing | SSRF via console_url / redirect | mitigate | validate_external_url(console_url) before any connection in enroll and push; httpx follow_redirects=False; _NoRedirectHandler imported for urllib fallbacks |
| T-108-05 | Info Disclosure | enrollment token leak to disk | mitigate | raw one-time token printed once, never written to sensor.yaml; only persistent hmac_key + console_api_token (push creds, distinct from the one-time token) are stored |
| T-108-06 | Repudiation | push payload replay | mitigate | payload_id is a unique uuid4 per push → console dedup → 409 (Phase 109); 409 treated as delivered success here |
| T-108-07 | Denial of Service | unbounded spool disk fill | mitigate | _evict_if_full enforces max-file-count AND max-total-bytes with oldest-evicted policy |
| T-108-08 | Tampering | path traversal in spool filenames | mitigate | filenames are {uuid4}.json.zst — UUID format, no operator-controlled path components |
</threat_model>
