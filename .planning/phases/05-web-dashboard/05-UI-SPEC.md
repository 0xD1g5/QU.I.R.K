## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, table cell dividers |
| sm | 8px | Compact row padding, chip spacing, inline element gaps |
| md | 16px | Default card padding, form field spacing, button padding |
| lg | 24px | Section header spacing, card-to-card gaps |
| xl | 32px | Panel padding, sidebar section breaks |
| 2xl | 48px | Page-level vertical rhythm between major sections |
| 3xl | 64px | Top-of-page hero area (score gauges section) |

Exceptions:
- Touch targets for icon-only buttons: minimum 44px × 44px (accessibility requirement)
- Cytoscape.js canvas: full remaining viewport height after nav — no fixed spacing token applies
- Score gauge ring diameter: 120px (fixed — not a spacing token)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.1 |

Notes:
- Body (14px/400) used for: table cells, finding descriptions, CBOM viewer text, sidebar nav items
- Label (12px/500) used for: badges, severity chips, column headers, metadata tags, confidence text
- Heading (20px/600) used for: section titles ("Executive Summary", "Findings", "CBOM Viewer")
- Display (28px/600) used for: overall readiness score number, page title in print view
- Monospace exception: algorithm names in CBOM table and finding detail use `font-mono` at 13px/400 (shadcn default mono stack)

## Color

This phase ships a dark-first security tool aesthetic. Phase 7 (Polish and Packaging) will refine
the visual identity. These tokens are intentionally neutral to allow Phase 7 to layer branding on top
without rework.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(240 10% 4%)` — Zinc 950 | Page background, full-bleed canvas areas, Cytoscape.js graph background |
| Secondary (30%) | `hsl(240 6% 10%)` — Zinc 900 | Cards, sidebar, nav bar, table row background, modal overlay |
| Accent (10%) | `hsl(210 100% 56%)` — Sky 500 | Reserved — see list below |
| Destructive | `hsl(0 72% 51%)` — Red 600 | Destructive action buttons only (no current destructive actions in Phase 5) |

Accent (`hsl(210 100% 56%)`) is reserved for:
1. Overall readiness score ring stroke (the arc gauge outline for the top-level score only)
2. "Export PDF" primary CTA button background
3. Active navigation tab underline/indicator
4. External link icons in findings detail
5. Focus ring on interactive form controls (search input, filter dropdowns)

Do NOT use accent for: severity badges, quantum-safety indicators, graph node colors, score sub-gauge rings.

**Semantic color tokens (in addition to the 60/30/10 split):**

| Semantic Role | Value | Used For |
|---------------|-------|----------|
| Quantum Safe | `hsl(142 71% 45%)` — Green 500 | Cytoscape nodes, CBOM quantum-safety badge (Safe) |
| Quantum At Risk | `hsl(38 92% 50%)` — Amber 500 | Cytoscape nodes, CBOM quantum-safety badge (At Risk) |
| Quantum Vulnerable | `hsl(0 72% 51%)` — Red 600 | Cytoscape nodes, CBOM quantum-safety badge (Vulnerable), severity CRITICAL badge |
| Severity HIGH | `hsl(24 95% 53%)` — Orange 500 | Severity badge background |
| Severity MEDIUM | `hsl(38 92% 50%)` — Amber 500 | Severity badge background |
| Severity LOW | `hsl(213 94% 68%)` — Blue 400 | Severity badge background |
| Severity INFO | `hsl(240 5% 46%)` — Zinc 500 | Severity badge background |
| Muted text | `hsl(240 5% 65%)` — Zinc 400 | Secondary labels, metadata, disabled states |
| Border | `hsl(240 6% 17%)` — Zinc 800 | Table dividers, card borders, input borders |

Source: CONTEXT.md D-13/D-14 (quantum-safety node colors), REQUIREMENTS.md UI-02 (severity breakdown)

## Layout Structure

### Navigation
- Left sidebar, fixed, 240px wide
- Sections: Executive Summary | Findings | Certificates | CBOM Viewer | Migration Roadmap
- Active item: accent underline + text at full opacity; inactive: muted text (Zinc 400)
- Sidebar collapses to icon-only (48px) on viewport < 1024px wide

### Main Content Area
- Width: `calc(100vw - 240px)` on desktop; full width on mobile
- Max content width: 1200px centered within main area
- Padding: xl (32px) horizontal, lg (24px) vertical on desktop; md (16px) on mobile

### Print Layout (`/print` route)
- No sidebar, no nav, no interactive controls
- Single column, full width
- Page breaks: CSS `break-before: page` before each major section
- Background: white (#ffffff), text: black (#0a0a0a), borders: #e4e4e7

## Screen-by-Screen Interaction Contract

### 1. Executive Summary (default route `/`)

**Score Gauges Row**
- 5 arc gauges in a horizontal row: Overall Readiness (larger, center) + 4 sub-scores (Hygiene, Modern TLS, Identity, Agility)
- Overall gauge: 160px diameter, accent ring stroke, score number in Display type (28px/600)
- Sub-gauges: 120px diameter, neutral Zinc 600 ring stroke (unfilled), colored fill by score range: ≥80 Green 500 / 50–79 Amber 500 / <50 Red 600
- Gauge labels: Label type (12px/500), below each gauge
- Confidence badge: inline next to Overall score label — pill shape, text matches quantum-safety label palette
- On hover: Tooltip showing raw score value and driver count

**Severity Breakdown Bar Chart**
- Horizontal bar chart, 5 bars: CRITICAL / HIGH / MEDIUM / LOW / INFO
- Bar colors: semantic severity tokens (above)
- Count label at end of each bar, right-aligned
- Chart height: 180px fixed; width: fills card
- Sourced from findings list severity counts

**Scan Metadata Row**
- Below gauges: scan timestamp, total endpoints scanned, total findings count
- Body type (14px/400), muted text color

### 2. Findings Table (route `/findings`)

- Full-width table, 8 columns: Severity | Host | Port | Title | Protocol | Quantum Risk | Source | Detail
- Default sort: Severity descending (CRITICAL first)
- Search: `input` component, top-right of table, filters Title and Host columns (client-side)
- Severity filter: `select` dropdown, multi-value, top-left of table
- Row expand: clicking anywhere on row opens `Sheet` slide-out from right
  - Sheet width: 480px
  - Sheet content: full finding title, host:port, severity badge, description, remediation, CVSS if present, quantum context
- Pagination: 25 rows per page, shadcn Pagination component at table bottom
- Empty state: see Copywriting Contract

### 3. Certificate Inventory (route `/certificates`)

- Table, 7 columns: Host | Port | Subject CN | Issuer | Expiry | Algorithm | Quantum Safety
- Default sort: Expiry ascending (soonest expiring first)
- Expiry cell: red text + warning icon if < 30 days; amber if < 90 days; normal muted text otherwise
- Quantum Safety column: Badge component with quantum-safety semantic colors
- No pagination on first render (certificate counts expected < 500 for a consulting scan)
- Empty state: see Copywriting Contract

### 4. CBOM Viewer (route `/cbom`)

**Table Tab (default)**
- Columns: Algorithm | Type | Key Size | Quantum Safety | Source Systems
- Source Systems column: comma-separated host:port or file paths, truncated at 3 with "+N more" overflow text
- Quantum Safety: Badge with semantic colors
- Sortable by all columns; filterable by Type and Quantum Safety
- Algorithm names: `font-mono` 13px

**Graph Tab**
- Cytoscape.js canvas, fills remaining viewport height (min 400px)
- Node types: Algorithm (circle, colored by quantum-safety) and System (rounded rectangle, Zinc 700)
- Edge: undirected, stroke weight proportional to frequency (1px = 1 system, max 6px)
- Layout: cose-bilkent force-directed; if node count < 15 use breadthfirst with algorithm nodes as roots
- Node tooltip on hover: algorithm name + quantum-safety label + usage count
- Zoom controls: +/- buttons (top-right of canvas), fit-to-screen button
- No pan lock — free pan and zoom

### 5. Migration Roadmap (route `/roadmap`)

- Cytoscape.js directed graph (DAG layout: dagre if available, breadthfirst fallback)
- Node: roadmap phase item — rectangle, label = phase title, color by timeframe: Immediate = Red 600, Short-term = Amber 500, Long-term = Green 500
- Edge: directed arrow, label = dependency reason (if roadmap data provides it)
- Canvas fills viewport height minus nav
- Static render (no interactivity beyond pan/zoom) — sourced from `build_phased_roadmap()` return value
- Node click: tooltip showing full `why` text from roadmap item

### 6. Print Layout (`/print`)

- Rendered by Playwright — not for direct human navigation
- Sections in order: Cover (scan date, target), Executive Summary (scores), Findings (full table, no pagination), Certificate Inventory, CBOM Table, Migration Roadmap (text list, not graph)
- No Cytoscape graphs in print — replace with structured text lists
- No interactive controls rendered
- Font: same Inter stack; background white; text black

## Accessibility Contract

- All interactive elements: keyboard navigable in DOM order
- Focus ring: 2px solid accent color, 2px offset
- Color: never convey meaning by color alone — severity badges include text label, quantum-safety badges include text label
- ARIA: Cytoscape canvas gets `role="img"` and `aria-label` describing the graph content
- Table: `<th scope="col">` on all column headers; sortable columns include `aria-sort` attribute
- Minimum contrast ratio: 4.5:1 for all body text on its background (WCAG AA)
