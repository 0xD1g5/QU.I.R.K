<objective>
Stand up a tag-triggered GHCR multi-arch container release path that publishes `ghcr.io/0xd1g5/quirk:{latest,vX.Y.Z}` for `linux/amd64` and `linux/arm64`. The image installs QU.I.R.K. from the PyPI release the existing `release.yml` workflow just published (not from the local source tree) — keeping the container a faithful reflection of the published wheel.

Purpose: A first-time evaluator can `docker run ghcr.io/0xd1g5/quirk:latest` on macOS arm64, Linux amd64, or Linux arm64 and get the CLI help with zero local Python setup.

Output: `Dockerfile`, `.dockerignore`, `.github/workflows/release-container.yml`, and a "Container Image" subsection appended to `docs/release-process.md`.
</objective>

- Default CMD: `quirk --help`.
- Multi-arch: `linux/amd64,linux/arm64` via `docker/build-push-action@v6` + `docker/setup-qemu-action@v3`.
- Registry: `ghcr.io/0xd1g5/quirk` (GHCR lowercase; namespace from existing repo URL `0xD1g5`).

Choice point resolution (CONTEXT.md "Claude's Discretion"): use a NEW sibling workflow `release-container.yml` rather than extending `release.yml`. Reason: (a) keeps the PyPI publish workflow's blast radius narrow — a container build failure must not block PyPI artifact publish; (b) the container workflow has different permissions (`packages: write` for GHCR vs `id-token: write` for OIDC PyPI); separating files makes the permission surfaces auditable.

    Shape:
    - `on.push.tags: ['v*.*.*']`
    - One job `container` on `ubuntu-latest` with `permissions: { contents: read, packages: write }`.
    - Steps:
      1. `actions/checkout@v4`
      2. Compute version: extract from `${{ github.ref_name }}` (strip leading `v`) into a step output `version`.
      3. **Wait for PyPI artifact availability** — poll `https://pypi.org/pypi/qu-i-r-k/${version}/json` with `curl -fsS` in a retry loop (max 20 attempts, 30s sleep) so the container build only proceeds once the PyPI publish job (in `release.yml`) has completed. This avoids the cross-workflow `needs:` limitation.
      4. `docker/setup-qemu-action@v3` (arm64 emulation).
      5. `docker/setup-buildx-action@v3`.
      6. `docker/login-action@v3` against `ghcr.io` using `${{ github.actor }}` + `${{ secrets.GITHUB_TOKEN }}`.
      7. `docker/build-push-action@v6` with:
         - `context: .`
         - `platforms: linux/amd64,linux/arm64`
         - `push: true`
         - `tags: |\n  ghcr.io/0xd1g5/quirk:latest\n  ghcr.io/0xd1g5/quirk:v${{ steps.ver.outputs.version }}`
         - `build-args: QUIRK_VERSION=${{ steps.ver.outputs.version }}`
         - `provenance: true`
    - Header comment documents the one-time GHCR setup: the GHCR package must be publicly visible (`Settings → Packages → quirk → Change visibility`) on first publish; subsequent publishes inherit.
