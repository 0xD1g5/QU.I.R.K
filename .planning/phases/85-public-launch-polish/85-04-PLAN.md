  <action>
    For each of the four fixtures, execute the following sequence (one fixture at a time — DO NOT parallelize the chaos lab brings-up; they share ports and state):

    1. `cd quantum-chaos-enterprise-lab && ./lab.sh down` — clean slate.
    2. Bring up the source profile(s) per the <interfaces> reconciliation table:
       - `tls-only.cbom.json`: bring up the profile(s) that produce weak-TLS findings. Per <interfaces>, start with `tls-cert-defects` profile; if that profile alone is insufficient (no findings emitted), fall back to default phaseA bring-up.        - `identity.cbom.json`: `./lab.sh up --profile smime --profile adcs`
       - `data-at-rest.cbom.json`: `./lab.sh up --profile database`
       - `data-in-motion.cbom.json`: `./lab.sh up --profile email --profile broker` plus the weak-TLS source from the tls-only fixture
    3. Wait for services healthy (use `./lab.sh status` until all relevant services are green).
    4. Run `quirk run` with the appropriate config pointing at the lab targets; emit CBOM JSON to a temp path.
    5. Post-process with `jq` to normalize timestamps, serial numbers, and UUIDs to deterministic placeholders; pretty-print at 2-space indent; write to `examples/cbom/<fixture>.cbom.json`.
    6. `./lab.sh down` before moving to the next fixture.

    Constraints:
    - Each output file MUST be ≥ 500 bytes (rough size sanity — no near-empty fixtures).
    - Each output file MUST contain at least one `"components"` entry with a `"type": "cryptographic-asset"` key (CycloneDX CBOM marker).
    - Files MUST be pretty-printed (`jq --indent 2` output, trailing newline).
    - Each MUST validate as JSON (`jq empty <file>` exits 0).
    - Filenames per D-LAUNCH Sample CBOMs: `{profile}.cbom.json` (already declared in `files_modified`).

    Phase 84 anti-pattern: do NOT `git add -A` after generation — explicit `git add examples/cbom/*.cbom.json` (the four declared files only).

    Per `CLAUDE.md` Chaos Lab Maintenance rule: this plan only consumes existing chaos lab profiles; it does NOT add/rename profiles. No lab.sh or expected_results_v4.md updates required.
  </action>
