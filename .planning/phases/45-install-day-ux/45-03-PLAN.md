<objective>
Make `coverage_gap` advisories visible in the HTML report and dashboard, while ensuring they
have ZERO impact on the readiness/intelligence score (D-07). Three integration points:

1. **Risk engine** (`quirk/engine/risk_engine.py`): branch on ADVISORY-protocol endpoints with
   `scan_error_category="missing_extra"` and emit a finding dict with
   `category="coverage_gap"`, `severity="INFO"`, and `recommendation` carrying the install hint.
2. **HTML report** (`report.html.j2` + `html_renderer.py`): add a "Coverage Gaps" section
   before "All Findings"; filter the All Findings loop to exclude coverage_gap; compute
   `sev_counts` over non-coverage_gap findings only.
3. **Score exclusion** (`evidence.py`): filter `coverage_gap` findings out of `finding_list`
   before computing `finding_severity_counts` and `totals.findings` — D-07's
   "no impact on readiness or confidence subscore" requirement.
4. **Dashboard DTO** (`schemas.py`): add `category: Optional[str] = None` to `FindingItem`
   (Q2 user decision; additive; no DB migration).

Plus the mandatory Phase 45 closing tasks: UAT-SERIES.md update + sync, Obsidian phase note,
final docs commits.
