    Create quirk/dashboard/api/middleware/sensor_auth.py with `require_sensor_auth(request: Request, credentials: Optional[HTTPAuthorizationCredentials] = Depends(_bearer), db: Session = Depends(get_db)) -> None`. Instantiate a module-level `_bearer = HTTPBearer(auto_error=False)` (mirror auth.py L31). Inject `db` so the 401 branches can write audit rows (D-09; RESEARCH Pitfall 2). Compute a surrogate `scan_id` from `datetime.now(timezone.utc).replace(tzinfo=None).strftime("%Y-%m-%dT%H:%M:%SZ")`.

    Define a nested `_audit_and_raise(status_code, error_summary, detail)` that adds an `IntegrationDelivery(scan_id=scan_id, finding_hash=None, destination="sensor_push", status="failed", attempted_at=now, error_summary=error_summary)` row, commits (guarding commit failures with `logger.warning(..., safe_str(exc))`), then raises `HTTPException(status_code, detail=detail)`.

    Flow: `credentials is None` → _audit_and_raise(401, "missing_sensor_token", "Sensor authentication required"). Else `hashed = hashlib.sha256(credentials.credentials.encode()).hexdigest()`; look up `db.query(SensorToken).filter(SensorToken.token_hash == hashed).first()`. None → _audit_and_raise(401, "unknown_sensor_token", "Unknown sensor token"). Use `hmac.compare_digest(hashed, token_row.token_hash)` as defense-in-depth on the matched row; mismatch → same unknown_sensor_token 401 (D-03 timing-safe discipline). `token_row.revoked_at is not None` → _audit_and_raise(401, "revoked_sensor_token", "Sensor token revoked"). On success: `request.state.sensor_id = token_row.sensor_id`. Never log `credentials.credentials`; copy the "Token never logged (T-102-09)" docstring annotation from auth.py. PEP 8, minimal diff.
