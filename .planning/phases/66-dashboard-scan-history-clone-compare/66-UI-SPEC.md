- Checkbox touch target: 20px × 20px (Radix CheckboxRoot default — do not override)

## Typography

All type roles use the Tailwind utility classes; no custom CSS needed unless noted.

| Role | Size | Weight | Line Height | Class / Notes |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` — table rows, finding descriptions |
| Label | 12px | 600 (semibold) | 1.4 | `text-xs font-semibold` — column headers, section labels, badge text |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` — page titles (`/scans`, `/compare`) |
| Display | 28px | 700 (bold) | 1.1 | `text-2xl font-bold font-data stat-metric` — numeric score values in compare header |

Data rule: any numeric value (score, delta, finding count) must use `font-data` class (JetBrains Mono, `tnum` feature) so digits align in tabular context.

Eyebrow labels (section group headers in compare view): use `.label-eyebrow` utility class from `index.css` — 10px / weight 600 / ALL CAPS / `letter-spacing: 0.08em`.

## Color

All values resolve via CSS custom properties. Dark-mode (Obsidian Pro) is the only shipped mode for Phase 66.

| Role | CSS Token | Computed (dark) | Usage |
|------|-----------|-----------------|-------|
| Dominant (60%) | `--ds-bg-page` | `#080a0f` | Page shell background |
| Secondary (30%) | `--ds-bg-base` / `--ds-bg-surface` | `#0d0f14` / `#161921` | Cards, table rows, sidebar, nav |
| Accent (10%) | `--ds-accent` | `#4ba8a8` | Reserved — see list below |
| Positive delta | `--ds-ok` | `#3fa876` | Score improvement indicator (Δ > 0) |
| Negative delta | `--ds-critical` | `#e05555` | Score regression indicator (Δ < 0) |
| Warning/notice | `--ds-high` | `#d4893a` | Amber reconstruction notice text |
| Warning bg | `--ds-high-dim` + `--ds-high-bdr` | `rgba(212,137,58,0.10)` / border | Amber notice card background |
| Destructive | `--destructive` | `hsl(0 68% 60%)` | No destructive actions in Phase 66 — not used |

Accent (`--ds-accent` #4ba8a8) is reserved for exactly these elements:

1. "Compare" sticky button (primary action) — `bg-primary` via shadcn Button default
2. Checkbox checked state — Radix default, no override
3. Active tab indicator underline on the `/compare` Tabs component
4. Focus ring (`--ring: 180 37% 47%`) on interactive controls

Accent MUST NOT be used for: table row hover, section separators, score values, or badge backgrounds.

Severity badge colors follow the established `SEVERITY_STYLES` map from `findings.tsx`:

| Bucket | Background | Text |
|--------|-----------|------|
| HIGH (CRITICAL+HIGH) | `--severity-high` / `--ds-high` | white |
| MEDIUM | `--ds-medium` | `--ds-bg-page` |
| LOW | `--severity-low` / `--ds-accent` | white |

Use `.severity-high-chip` / `.severity-ok-chip` / `.severity-medium-chip` / `.severity-critical-chip` utility classes from `index.css` for finding row chips.

## Interaction Contracts

### Checkbox Selection (two-scan window)

- Each row has a `<Checkbox>` bound to local state `Set<string>` of selected scan IDs.
- When a 3rd checkbox is checked: remove the oldest entry from the set before adding the new one (FIFO window of 2). No alert or toast — the oldest checkbox visually unchecks.
- Compare button disabled state: `selectionCount !== 2` → `<Button disabled>`.
- Compare button enabled state: navigate to `/compare?a=<id>&b=<id>` — newer scan as `a`, older scan as `b` (descending timestamp order).

### Clone Button

- On click: navigate to `/scan/new?target=<encoded>&profile=<profile>&calibration=<calibration>`.
- If the scan is CLI-launched (no `scan_jobs` row): the `/scan/new` page renders an amber notice card above the Targets field. The notice uses `--ds-high-dim` background + `--ds-high-bdr` border + `--ds-high` text color.
- No modal or confirmation required for clone — it is non-destructive navigation.

### Tabs (Compare Page)

- Default open tab: **Findings** (index 0) — sourced from CONTEXT.md §Claude's Discretion.
- Tab labels include counts: "Findings (3)" / "Subscores (6)" / "Endpoints (2)".
- Empty tab content: show `<EmptyStateCard>` inline (do not hide the tab or disable the trigger).

### Loading States

- `/scans` initial load: `<PageSpinner>` until first `/api/scans` response.
- `/compare` initial load: `<PageSpinner>` until `/api/compare` response.
- After data arrives: render table / diff view. No skeleton rows for these pages (data arrives as a single fetch).

### Error States

- `/scans` fetch error: render an inline error card (no `<PageSpinner>`), with the error copy from the Copywriting Contract below.
- `/compare` 400 (a === b): render inline error card, not a toast.
- `/compare` other error: render inline error card with generic error copy.

## Score Delta Visual Contract

| Condition | Badge text | Badge color |
|-----------|-----------|-------------|
| scan_a.score > scan_b.score | +{N} pts improvement | `--ds-ok` (#3fa876) background, white text |
| scan_a.score < scan_b.score | −{N} pts regression | `--ds-critical` (#e05555) background, white text |
| scan_a.score === scan_b.score | No score change | `variant="outline"` / `text-muted-foreground` |

Arrow glyphs: use Lucide `TrendingUp` / `TrendingDown` icons (16px, inline) next to delta badge in the compare header. Do not use unicode arrows (▲/▼) in the header — reserve those for score delta Badge in table rows (consistent with existing `ScoreDeltaBadge` in `trends.tsx`).
