    pip install --upgrade 'quirk[motion]'

    ## Known Limitations

    - **Port 25 cloud egress.** Many cloud providers block outbound SMTP on port 25; a scan
      run from such a host will return `CONNECTION_REFUSED` for that port. The scanner handles
      this gracefully (no crash, finding emitted as the absent-or-blocked category).
    - **STARTTLS stripping detection is static.** sslyze cannot detect an active stripping
      attack agentlessly; QU.I.R.K. emits a STARTTLS-downgrade-risk MEDIUM on every port-25
      endpoint that successfully negotiates TLS, on the principle that the *capability* is
      present even when no active attack is observed.
    - **KAFKA-04 enrichment is optional.** AdminClient describe_configs requires
      `quirk[kafka]` and authenticated API access. When the library is absent or auth fails,
      the scanner falls back to sslyze-only TLS posture (no functional regression).
    - **Azure Service Bus and AWS SQS probes test the TLS layer only.** No SAS-token or
      AWS-credential authentication is required or attempted.
