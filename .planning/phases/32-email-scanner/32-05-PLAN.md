**`labs/email/Dockerfile`** — single image hosting both Postfix and Dovecot (D-15 says decoupled certs; one container is fine — split is optional). Use `ubuntu:22.04` per EMAIL-11. Install postfix + dovecot-imapd + dovecot-pop3d + openssl + supervisor:

```dockerfile
FROM ubuntu:22.04

ARG DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y --no-install-recommends \
        postfix \
        dovecot-imapd \
        dovecot-pop3d \
        openssl \
        ca-certificates \
        supervisor \
    && rm -rf /var/lib/apt/lists/*

# Postfix needs a user and minimal queue dirs; dpkg postinst handles most of this.
COPY postfix/main.cf   /etc/postfix/main.cf
COPY postfix/master.cf /etc/postfix/master.cf
COPY dovecot/dovecot.conf  /etc/dovecot/dovecot.conf
COPY dovecot/10-ssl.conf   /etc/dovecot/conf.d/10-ssl.conf

# Cert volumes mounted at runtime: /etc/postfix/certs/ and /etc/dovecot/private/
RUN mkdir -p /etc/postfix/certs /etc/dovecot/private

# Supervisor runs both daemons in the same container
RUN echo '[supervisord]\nnodaemon=true\n\n[program:postfix]\ncommand=/usr/sbin/postfix start-fg\nautorestart=true\n\n[program:dovecot]\ncommand=/usr/sbin/dovecot -F\nautorestart=true\n' > /etc/supervisor/conf.d/email-lab.conf

EXPOSE 25 465 587 110 143 993 995

CMD ["/usr/bin/supervisord", "-c", "/etc/supervisor/supervisord.conf"]
```

(If splitting into separate Postfix and Dovecot containers is cleaner per the team's preference, that is acceptable — the compose file in Task 2 reflects the choice. Go with the single-container approach unless the existing `labs/vault/Dockerfile` precedent argues otherwise; verify before deciding.)

**`labs/email/Makefile`** — per 32-PATTERNS.md lines 661–678 verbatim, with these refinements:

```makefile
.PHONY: certs clean help

CERTS_DIR := certs

help:
	@echo "labs/email targets:"
	@echo "  make certs   - regenerate self-signed RSA-2048 certs (D-15/D-16)"
	@echo "  make clean   - remove generated certs"

certs:
	@mkdir -p $(CERTS_DIR)
	openssl req -x509 -newkey rsa:2048 -keyout $(CERTS_DIR)/postfix.key \
	    -out $(CERTS_DIR)/postfix.crt -days 3650 -nodes \
	    -subj "/CN=postfix.chaos.local" 2>/dev/null
	openssl req -x509 -newkey rsa:2048 -keyout $(CERTS_DIR)/dovecot.key \
	    -out $(CERTS_DIR)/dovecot.crt -days 3650 -nodes \
	    -subj "/CN=dovecot.chaos.local" 2>/dev/null
	chmod 644 $(CERTS_DIR)/*.crt
	chmod 600 $(CERTS_DIR)/*.key
	@echo "Generated $(CERTS_DIR)/{postfix,dovecot}.{crt,key}"

clean:
	rm -f $(CERTS_DIR)/postfix.* $(CERTS_DIR)/dovecot.*
```

**`labs/email/postfix/main.cf`** — weak TLS config per 32-RESEARCH.md lines 595–614:

```ini
# Phase 32 chaos lab — intentionally weak TLS for scanner detection
myhostname = postfix.chaos.local
mydomain = chaos.local
inet_interfaces = all
inet_protocols = ipv4
mynetworks = 0.0.0.0/0

# TLS protocol pinning — TLS 1.0/1.1 disabled by OpenSSL 3.x on scanner host anyway
smtpd_tls_protocols = !SSLv2, !SSLv3
smtp_tls_protocols = !SSLv2, !SSLv3

# Force RSA key exchange / non-PFS by excluding ECDHE/EDH/DHE
smtpd_tls_mandatory_ciphers = medium
smtpd_tls_mandatory_exclude_ciphers = aNULL, eNULL, EXPORT, DES, MD5, PSK, SRP, CAMELLIA, ECDHE, EDH, DHE
smtpd_tls_ciphers = medium
smtpd_tls_exclude_ciphers = aNULL, eNULL, EXPORT, DES, MD5, PSK, SRP, CAMELLIA, ECDHE, EDH, DHE

# Enable STARTTLS on port 25 + 587; implicit TLS on 465 (configured via master.cf)
smtpd_tls_security_level = may
smtpd_use_tls = yes
smtpd_tls_cert_file = /etc/postfix/certs/postfix.crt
smtpd_tls_key_file  = /etc/postfix/certs/postfix.key

# Loglevel for chaos-lab debugging
smtpd_tls_loglevel = 1
```
