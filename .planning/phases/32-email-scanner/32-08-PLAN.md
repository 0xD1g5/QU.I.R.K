# Plan 32-08 — Email Scan JSON Persistence (Gap Closure)

## Goal

Make `CryptoEndpoint.email_scan_json` non-NULL on at least one endpoint per
scanned host, holding a structured summary of all 7 port scans for that host.
Mirror the `kerberos_scan_json` attachment pattern in
`quirk/scanner/kerberos_scanner.py:294,329,340`.

## Requirements covered

- **EMAIL-11** (DB persistence of `email_scan_json`)
- Phase 32 SC-1 ("scan results accessible in DB `email_scan_json` column")

## Tasks

1. Write a failing test in `tests/test_email_scanner.py`:

   ```python
   def test_email_scan_json_attached_per_host():
       """Phase 32 SC-1: at least one endpoint per host has email_scan_json populated."""

                 "scan_error": getattr(ep, "scan_error", None),
                 "tls_blocker_reason": getattr(ep, "tls_blocker_reason", None),
             }
             for ep in host_endpoints

     }
     ```

     for that host (sort by port for determinism, matching kerberos pattern).
