## Caveats

### OpenSSL 3.x TLS 1.0/1.1 limitation

The scanner host's OpenSSL 3.x library refuses to negotiate TLS 1.0 or TLS 1.1 even when the server offers them. This lab is configured `ssl_min_protocol = TLSv1.2` to keep behavior deterministic — TLS-1.1 detection is NOT a Phase 32 expectation. (See `.planning/phases/32-email-scanner/32-RESEARCH.md` "Pitfall 3".)

### Port 25 cloud egress

If the lab is run on a cloud VM where outbound TCP 25 is blocked, the scanner reports `tls_blocker_reason: CONNECTION_REFUSED` for that endpoint and does not emit findings — this is correct behavior per D-03.

### Layered findings on port 30025

A single port-30025 endpoint emits BOTH a MEDIUM downgrade-risk finding AND a HIGH weak-cipher finding (per D-11). The dashboard's `_dedupe_findings` does NOT collapse these because they have distinct titles. This is intentional.
