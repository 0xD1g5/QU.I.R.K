<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| spec source (URL/$ref) → scanner | Untrusted URLs and $ref values can target internal network (SSRF) |
| spec bytes → YAML parser | Untrusted spec content can be a decompression bomb (DoS) |
| pip resolver → [all] extras | A transitive dep could silently pull a dangerous package |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-94-06 | Information Disclosure | URL fetch scope bypass (SPEC-02) | mitigate | Scope gate (url.startswith(target)) raises before any request; then validate_external_url blocks metadata/private IPs always |
| T-94-07 | Denial of Service | oversized / billion-laughs YAML | mitigate | 10MB raw-byte gate BEFORE yaml.safe_load on both file and URL paths |
| T-94-08 | Information Disclosure | spec content / URL in error messages | mitigate | SpecParsingError messages use _redact_preview; exceptions wrapped in safe_str() before log |
| T-94-SC | Tampering | pip install openapi-spec-validator | mitigate | RESEARCH Package Legitimacy Audit verdict [OK] (8-yr-old PyPI package); [api] excluded from [all]; CI guard test enforces schemathesis + [api] absence from [all] |
