## Spacing Scale

Declared values (must be multiples of 4). Inherits project 8-point scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, radio gap-to-label |
| sm | 8px | Compact element spacing; gap between radio options in a group |
| md | 16px | Default element spacing; card padding; question text margin from number |
| lg | 24px | Section padding; gap between question cards within a practice area |
| xl | 32px | Gap between practice area sections within a tab |
| 2xl | 48px | Tab content top padding; major section breaks in Org Profile wizard |
| 3xl | 64px | Page-level vertical padding |

Exceptions:
- Touch targets on nav items: 44px min-height (already enforced via `min-h-[44px]` in sidebar.tsx — maintain this for wizard step buttons)
- Evidence note textarea: min-height 80px (2 lines), not constrained to 8-point multiples

## Typography

All sizes are Tailwind scale equivalents. Body text uses system UI (shadcn default).
Data values (scores, percentages, raw numbers) use JetBrains Mono via `.font-data` or `.stat-metric` utility classes from index.css.

| Role | Tailwind Class | Size | Weight | Line Height | Usage |
|------|---------------|------|--------|-------------|-------|
| Body | `text-sm` | 14px | 400 (regular) | 1.5 | Question text, evidence note placeholder, wizard field labels, table cells |
| Label | `text-xs` | 12px | 600 (semibold) | 1.4 | Table header (`font-semibold`), badge text, maturity level radio labels, section subheadings, progress counter |
| Heading | `text-base` | 16px | 600 (semibold) | 1.3 | Practice area section headers, wizard step titles, scorecard panel headings |
| Display | `text-lg` | 18px | 700 (bold) | 1.2 | Page-level headings ("QRAMM Assessment", "Org Profile"), scorecard dimension score values |

Eyebrow labels (10px / semibold / all-caps / 0.08em tracking): use `.label-eyebrow` utility class from index.css. Apply to: tab group label above the 5 tabs, "PRACTICE AREA" section labels, scorecard "DIMENSION SCORES" column header.

Monospace numerics: apply `.font-data` to raw score, weighted score, benchmark, and completion percentage values in the dimension summary table.

## Color

All values reference CSS custom properties — never use hardcoded hex or hsl literals in new components. Both dark and light modes are supported via the existing `:root` / `.light` override in index.css.

| Role | CSS Variable | Resolved (dark) | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `hsl(var(--background))` | `#080a0f` | Page shell, full-width layout background |
| Secondary (30%) | `hsl(var(--card))` / `hsl(var(--secondary))` | `#0d0f14` / `#161921` | Question cards, wizard step cards, scorecard panels, sidebar nav |
| Accent (10%) | `hsl(var(--accent))` = teal `#4ba8a8` | `#4ba8a8` | Reserved-for list below |
| Destructive | `hsl(var(--destructive))` | `#e05555` | "New Assessment" confirmation dialog only |

**Accent (`hsl(var(--accent))`) is reserved exclusively for:**
1. Active tab indicator underline/border on the 5-tab assessment view
2. Active sidebar nav item left border + background tint (`bg-accent/10`)
3. Selected radio button ring/fill (RadioGroup indicator)
4. Confirm button (primary variant, uses `--primary` = `#4ba8a8`)
5. "Calculate Score" button (primary variant)
6. Focus ring on all interactive elements (`ring: hsl(var(--ring))` = teal)
7. Progress bar fill on per-dimension completion counters

**Semantic maturity-level color assignments (scorecard only):**
- Optimizing (4): `hsl(var(--quantum-safe))` = `#3fa876` (green)
- Established (3): `hsl(var(--severity-low))` = `#4ba8a8` (teal)
- Developing (2): `hsl(var(--quantum-at-risk))` = `#d4893a` (amber)
- Basic (1): `hsl(var(--quantum-vulnerable))` = `#e05555` (red)

These tokens already exist in tailwind.config.ts — no new tokens needed.

**Auto-fill badge styling:** use `.severity-accent-chip` utility class (teal dim background + teal border + teal text). When modified: use `.severity-medium-chip` (neutral). Both classes already defined in index.css.

**RadarChart colors:**
- Assessment radar fill: `hsl(var(--accent))` at 20% opacity (`rgba(75, 168, 168, 0.20)`)
- Industry benchmark radar fill: `hsl(var(--muted-foreground))` at 15% opacity
- Grid lines: `var(--ds-border)` (`#252d3e`)
- Axis labels: `var(--ds-text-muted)` (`#6e7a95`)

## Component Interaction Contracts

### Org Profile Wizard (`/qramm`)

**Layout:** Single-page wizard with 5 fields presented vertically in a Card. No step-by-step pagination — all fields visible at once. Submit button at bottom of card.

**Fields (all use Select component):**
1. Industry Sector — options: Financial Services, Healthcare, Government, Technology, Retail, Energy, Other
2. Organization Size — options: 1–50, 51–500, 501–5000, 5000+
3. Geographic Scope — options: Single Country, Multi-Country, Global
4. Data Sensitivity — options: Public, Internal, Confidential, Restricted / Secret
5. Regulatory Obligations — multi-select or comma-separated input: PCI-DSS, HIPAA, SOC2, ISO 27001, NIST CSF, CMMC, None

**Auto-resume state:** If a session exists and an org profile is saved, `/qramm` shows a "Resume Assessment" card overlay instead of the blank wizard form. This card shows: org name summary + "Continue Assessment" primary button + "New Assessment" destructive secondary button.

**"New Assessment" interaction:** Renders an inline confirmation (not modal) with copy: "Starting a new assessment will archive your current progress. This cannot be undone." + "Confirm New Assessment" (destructive button) + "Cancel" (ghost button).

### Assessment View (`/qramm/assessment`)

**5-tab layout:** Tabs component (`tabs.tsx`), full-width, tabs left-aligned. Tab order: CVI | SGRM | DPE | ITR | Scorecard.

**Dimension tab content:** 3 Collapsible sections per tab. All 3 expanded by default on tab load (D-08).

**Practice area section header:** `[Practice Area Name] — [X/10 answered]`
- Practice area name: `text-base font-semibold`
- Answer counter: `text-xs text-muted-foreground` (e.g., "3/10 answered")
- Chevron collapse icon: `lucide-react ChevronDown`, rotates 180deg when collapsed, transition 200ms

**Question card layout (Card component):**
```
┌─────────────────────────────────────────────────────┐
│ Q[N] — Question text                     [badge?]   │
│                                                     │
│ ○ Basic (1)   ○ Developing (2)                      │
│ ○ Established (3)   ○ Optimizing (4)                │
│                                                     │
│ Evidence note (optional):                           │
│ ┌─────────────────────────────────────────────────┐ │
│ │ textarea (min 80px height)                      │ │
│ └─────────────────────────────────────────────────┘ │
│                                [Confirm] (if badge) │
└─────────────────────────────────────────────────────┘
```
- Question number: `text-xs text-muted-foreground font-semibold` (e.g., "Q12")
- Question text: `text-sm` weight 400, line-height 1.5
- Radio options: RadioGroup with horizontal layout (2×2 grid on narrow, 1×4 row on wide)
- Radio label text: `text-xs` weight 400 (maturity label) + `text-xs text-muted-foreground` (numeric in parens)
- Evidence note label: `text-xs text-muted-foreground`
- Evidence textarea: `text-sm`, `placeholder="Add supporting evidence..."`, no resize handle

**Auto-fill badge:** `.severity-accent-chip` class, text: "Auto-filled from scan", placed top-right of question card header row.

**Modified badge:** `.severity-medium-chip` class, text: "Modified from scan suggestion", replaces auto-fill badge when radio selection changes from suggestion.

**Confirm button:** `Button` variant="default" (teal primary), size="sm", text: "Confirm Answer". Appears below the radio group, right-aligned within the card. Visible only when badge is shown. On click: writes `answer_value`, sets `confirmed_at`, removes badge.

**Per-dimension progress:** `Progress` component above each tab's collapsible list. Label: `text-xs text-muted-foreground` "X of 30 answered". Progress fill uses accent teal.

### Scorecard Tab

**Layout:** Two columns on desktop (≥1024px), stacked on mobile.
- Left column (60%): RadarChart (4-axis: CVI, SGRM, DPE, ITR)
- Right column (40%): Maturity distribution histogram

**Below columns (full width):** Dimension summary table.

**RadarChart spec:**
- Component: `recharts` `RadarChart` with `PolarGrid`, `PolarAngleAxis`, `Radar`
- Rendered as static SVG (no animation, no hover interactions) — required for PDF embed (QRAMM-16 downstream)
- Size: 320px × 320px fixed (center-aligned in its column)
- Two overlaid Radars: "Assessment" (teal fill) + "Industry Benchmark" (muted fill, dashed stroke)
- Benchmark radar omitted (axis shows "—") when no org profile sector is set

**Dimension summary table columns:**
| Dimension | Raw Score | Weighted Score | Industry Benchmark | Maturity Level | Completion % |
- "Raw Score" and "Weighted Score": `.font-data` (JetBrains Mono, tabular figures)
- "Industry Benchmark": `.font-data`, shows "—" if no org profile
- "Maturity Level": colored Badge using semantic maturity tokens above
- "Completion %": `.font-data`

**Maturity distribution:** 4 horizontal bar rows (Basic / Developing / Established / Optimizing), each showing count of practice areas at that level. Bar color matches semantic maturity token.

**"Calculate Score" button:** `Button` variant="default", text: "Calculate Score", full-width within a Card, placed above the RadarChart. Triggers `POST /api/qramm/sessions/{id}/score`. Loading state: button text changes to "Calculating..." with a `Loader2` spinner icon (lucide-react), button disabled during request.

**"New Assessment" button:** `Button` variant="outline" size="sm", text: "New Assessment", placed in scorecard tab header row (right-aligned). Same confirmation inline pattern as Org Profile page.

**Unscored state (before Calculate Score is clicked):** All score cells show "—". Maturity Level column shows "—". Radar chart renders with all axes at 0 (invisible or zero-radius polygon). Muted-foreground callout text below the chart: "Scores calculated on explicit action. Answer questions, then click Calculate Score."

## State and Interaction Contracts

### QRAMMContext shape (from CONTEXT.md D-14)

```
sessionId: string | null
profile: OrgProfile | null
answers: Map<questionNumber, {
  answer_value: 1|2|3|4|null,
  suggested_answer: 1|2|3|4|null,
  confirmed_at: ISO8601 | null,
  evidence_note: string
}>
scoreResult: ScoreResult | null
```

### Debounce contract (QRAMM-10)
- 300ms debounce on every answer change
- Triggers `POST /api/qramm/assessment/draft`
- No loading indicator during debounced saves (silent)
- On save error: show a non-blocking toast notification (bottom-right): "Answer not saved — check connection" — no modal, no blocking

### Loading states
- Page mount / session fetch: show `Skeleton` components in place of question cards (3 skeleton rows per practice area)
- Calculate Score: button spinner (see above), no skeleton — table cells update in-place

## Interaction Accessibility Contracts

- All radio groups use `RadioGroup` + `RadioGroupItem` from shadcn (Radix UI) — keyboard navigation (arrow keys between options) provided by Radix.
- Confirm button: `aria-label="Confirm answer for question {N}"` to distinguish per-question buttons from one another.
- Practice area Collapsible trigger: `aria-expanded={open}` (provided by Radix Collapsible).
- Tab panel: `role="tabpanel"` with `aria-labelledby` referencing tab trigger id (provided by Radix Tabs).
- Progress bar: `aria-label="X of 30 questions answered"` on the Progress component.
- Auto-fill badge: `role="status"` so screen readers announce it when it appears.
- RadarChart: static SVG — add `aria-label="QRAMM radar chart showing dimension scores"` + `role="img"` on the outer `<svg>`.
- All sidebar nav items: 44px min-height (already enforced in sidebar.tsx `min-h-[44px]`).
- Color is never the sole differentiator for maturity levels — maturity labels ("Basic", "Developing", etc.) always appear as text alongside color.
