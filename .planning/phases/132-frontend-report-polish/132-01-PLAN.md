Output: AuthProvider.tsx, api.ts, and security_headers.py updated; Plan 00's CSP and sessionStorage RED tests turn GREEN.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → dashboard API | Auth token crosses from SPA storage into the X-API-Key request header (untrusted client) |
| server → browser | API responses carry security headers that constrain client-side script execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-132-01-01 | Information Disclosure | quirk_api_token in browser storage | mitigate | Move token to sessionStorage so it is purged on tab close, shrinking the XSS token-exfiltration window vs persistent localStorage (AUDIT-14 / D-01) |
| T-132-01-02 | Tampering | Injected/inline scripts in SPA | mitigate | Add CSP `script-src 'self'; object-src 'none'` via _STATIC_HEADERS setdefault, applied to all responses; blocks inline + remote script injection (D-04/D-05) |
| T-132-01-03 | Spoofing | Downstream proxy header override | accept | `setdefault` never overwrites a header set upstream; a misconfigured proxy could weaken CSP, but that is operator-controlled and out of scope; preserved existing pattern |
| T-132-01-04 | Denial of Service | sessionStorage unavailable (private mode) | mitigate | try/catch guards fall back to in-memory state transition; auth still functions for the session without crashing |
| T-132-01-SC | Tampering | npm/pip installs | mitigate | No new packages added; uses existing deps only. No legitimacy gate required. |
</threat_model>
