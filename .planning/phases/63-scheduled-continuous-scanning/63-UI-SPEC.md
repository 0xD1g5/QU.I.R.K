## Spacing Scale

Declared values (all multiples of 4, following existing project convention):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Table cell inner padding, tight stack spacing |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card gap |
| xl | 32px | Layout row gaps |
| 2xl | 48px | Page-level top margin |
| 3xl | 64px | Major section breaks |

**Exceptions:**
- Switch/Toggle touch target: minimum 44px height (WCAG 2.5.5 AA). Use `min-h-[44px]` wrapper around the Switch + label row.
- Table row height: 48px (`py-3` = 12px × 2 top/bottom + 24px content), matching findings page convention.

*Source: existing codebase convention (findings.tsx, sidebar.tsx) + WCAG AA touch target.*

## Interaction Contracts

### Enable/Disable Toggle

- Component: `Switch` (shadcn, Radix primitive)
- Optimistic update: flip the `enabled` state in local React state immediately on user click, before the `PATCH` response returns (prevents the "bounce" pitfall documented in RESEARCH.md Pitfall 6)
- On PATCH success: state confirmed, no additional re-render needed
- On PATCH failure: revert the optimistic flip, show inline error toast or inline row error text (use `text-destructive text-xs` below the switch)
- ARIA label on Switch: `aria-label="{name} — {enabled ? 'enabled' : 'disabled'}"`
- Switch must not be interactive during an in-flight PATCH for the same row (set `disabled` prop while loading)

### Delete Schedule

- Trigger: Ghost icon Button with `Trash2` icon, `aria-label="Delete {name}"`
- Confirmation: `Dialog` with:
  - Title: "Delete schedule?"
  - Body: "This will permanently remove '{name}'. Running or pending dispatches will not be interrupted."
  - Actions: Cancel (secondary Button) + "Delete" (destructive Button using `variant="destructive"`)
- On DELETE success: remove row from local state optimistically
- On DELETE failure: show inline error in Dialog footer (`text-destructive text-sm`)

### Loading State

- Show `SchedulesSkeleton` (4 animated Skeleton rows, same pattern as `FindingsSkeleton`) while `loading === true`
- Skeleton rows must have the same column widths as the real table to prevent layout shift

### Error State

- On fetch error (`error !== null`): render `EmptyStateCard` with the error copy (see Copywriting below) instead of the table
- Do not show a blank page or throw — always render something

### Add Schedule (Phase 63 scope: link out to CLI)

- The "Add Schedule" Button in the page header opens a Sheet (right drawer) OR links to documentation — **this phase defers a form-based create UI to Phase 65**
- For Phase 63: the Button displays a Sheet explaining that schedules are managed via `quirk schedule add ...` CLI with a code block showing the example command from CONTEXT.md success criterion 1
- Sheet title: "Add a Schedule"
- Sheet body: `text-sm text-muted-foreground` explanation + `<pre className="bg-muted rounded-md p-4 font-mono text-sm">quirk schedule add --name "weekly-prod" --cron "0 2 * * 1" --target prod.example.com --profile balanced</pre>`

No third-party cron-to-human library; implement a simple switch for the 5–6 most common patterns (daily, weekly, hourly, every N hours), fall back to raw expr.
