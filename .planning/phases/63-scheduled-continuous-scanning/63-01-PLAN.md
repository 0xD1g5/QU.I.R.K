<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator shell → quirk CLI | Operator-supplied `--name`, `--cron`, `--target`, `--profile`, `--config` flags reach SQLAlchemy and croniter |
| quirk CLI → SQLite (quirk.db) | CLI writes new rows; existing DB file may be on a shared filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-01 | Tampering | `cron_expr` argparse input | mitigate | `croniter.is_valid(cron_expr)` rejects non-cron strings before any DB write or downstream subprocess use |
| T-63-02 | Tampering | `name` (used as path fragment in Plan 02 dispatcher) | mitigate | `unique=True` SQL constraint plus argparse string type; restrict to no path separators by validating `re.match(r"^[A-Za-z0-9_\-\.]{1,255}$", args.name)` before INSERT — reject `..` and `/` |
| T-63-03 | Information Disclosure | IntegrityError raw exception text | mitigate | Catch `IntegrityError` and emit a fixed message ("Schedule '{name}' already exists") — never `f"...: {exc}"` (LEAK-02 pattern from Phase 59) |
| T-63-04 | Denial of Service | Unbounded schedule count blowing up `list` | accept | Single-user CLI; operator self-throttles. No rate limiting in v4.8 scope. |
| T-63-05 | Elevation of Privilege | `--config` arbitrary path → write to attacker-controlled SQLite file | accept | `--config` is operator-supplied via their own shell; file system permission model already governs which DB they can open. |
</threat_model>
