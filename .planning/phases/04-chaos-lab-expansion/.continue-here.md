<decisions_made>

**LAB-01 (JWT):**
- 4 separate microservice containers: RS256, HS256-weak, RSA1024, alg:none
- Each exposes `/.well-known/jwks.json` + `/token` endpoint
- Language/framework: Claude's discretion (Python/FastAPI preferred)

**LAB-02 (Registry):**
- 3 custom Dockerfiles: `image-old-libssl` (ubuntu:18.04 + OpenSSL 1.0.x), `image-old-pycrypto` (python-cryptography<3.0 + pyOpenSSL<22), `image-mixed` (old libssl + old pycrypto + Go binary)
- Images pushed to Docker Registry v2 via seed script on startup

**LAB-03 (Source):**
- Gitea with init script on container startup (API-based, no pre-baked volume)
- 4 anti-pattern categories seeded across 2-3 repos in multiple languages (Python, Go, Java)

**LAB-04 (Storage):**
- New `storage` profile — independent of existing `cloud` profile
- LocalStack KMS-only + HashiCorp Vault dev mode + postgres/pgcrypto
- Vault sidecar writes transit keys and KV secrets on startup

**LAB-05 (ssh-weak):**
- Custom Dockerfile (ubuntu:20.04 base) with sshd_config enabling weak KEX (diffie-hellman-group1-sha1, diffie-hellman-group14-sha1), weak hostkey (ssh-dss), weak MAC (hmac-md5, hmac-sha1)
- Uses `ssh-weak` profile — separate from existing vanilla `ssh-alt` service

**LAB-06 (LDAPS):**
- New `ldaps` profile with independent `osixia/openldap` container
- LDAP_TLS=true, certs from existing `./certs/`, LDAPS on port 636
- Existing `identity` profile openldap (plain LDAP port 13890) untouched
