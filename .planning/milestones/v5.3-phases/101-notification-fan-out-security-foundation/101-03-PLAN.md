<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator-configured URL/SMTP host → outbound network connection | User-supplied destinations could target internal/metadata services (SSRF) |
| minimal install → optional slack_sdk import | A top-level import would break installs without [notify] |
| payload dict → third-party endpoint | Webhook body must not carry infra topology |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-06 | Elevation of Privilege | all three channels | mitigate | validate_external_url at delivery time before every connection; SMTP host validated via https://host:port form; metadata IP always blocked (ISEC-01) |
| T-101-07 | Tampering | slack.py import chain | mitigate | find_spec("slack_sdk") gate + lazy import inside dedicated send_slack helper; graceful skip with advisory WARNING (ISEC-04); shadow-trap-safe |
| T-101-08 | Information Disclosure | webhook.py body | mitigate | Body is to_integration_payload(report) output only (whitelist enforced upstream in Plan 02); test asserts no host/port keys |
| T-101-09 | Denial of Service | smtplib / urlopen | mitigate | Mandatory timeout on SMTP, SMTP_SSL, and urlopen so a hung endpoint never stalls the scheduler loop (Pitfall 3) |
</threat_model>
