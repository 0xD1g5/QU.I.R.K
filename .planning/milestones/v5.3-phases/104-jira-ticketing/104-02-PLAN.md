| env vars → JIRA client | credentials resolved at __init__; must never persist or log |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-104-06 | Info Disclosure | Jira auth in exceptions | mitigate | all errors flow through base.dispatch_finding safe_str; jira import error advisory carries no creds; Task 2 test_credentials_not_in_logs asserts (ISEC-02) |
| T-104-07 | Availability | module-scope jira import | mitigate | lazy `from jira import JIRA` inside __init__ only; test asserts import without [tickets] (ISEC-04) |
| T-104-08 | Tampering | JQL label/project injection | accept | fingerprint is 64-char hex; project key double-quoted; finding-derived data is not interpolated into JQL |
</threat_model>
