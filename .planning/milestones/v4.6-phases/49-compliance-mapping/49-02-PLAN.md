<rationale>
The previous draft of this plan walked through three supplanted normalization
strategies (trailing-paren regex → all-paren regex → all-paren + prefix
fallback). All three were incorrect for a different reason: they stripped
parens from titles where parens are LEGITIMATELY part of the canonical key
(e.g., `"Legacy TLS versions allowed (TLS 1.0/1.1)"` and
`"Plaintext Redis listener (no auth)"` are emitted verbatim with parens at
risk_engine.py:464 and the Redis emit site).

**Final decision (grounded by reading risk_engine.py):**

1. COMPLIANCE_MAP keys are the LITERAL titles emitted by risk_engine, parens
   preserved. No paren-stripping is performed at lookup time.
2. The 7 f-string titles whose runtime form contains an interpolated
   substring (version/label/name) cannot be COMPLIANCE_MAP keys directly.
   Instead, each is mapped via TITLE_PREFIX_ALIASES: a literal source-text
   PREFIX → its canonical COMPLIANCE_MAP key (or UNMAPPED_TITLES key).
3. `_normalize_for_compliance(title)` is a single-purpose function: longest
   prefix in TITLE_PREFIX_ALIASES wins → return alias; otherwise return title
   verbatim.
4. The aggregator in tests/fixtures/chaos_lab_findings.py applies the IDENTICAL
   function (lazy-importing TITLE_PREFIX_ALIASES). This guarantees the title-
   join gate is sound.

This is the minimum-surface design. Everything else is ceremony.
</rationale>

    def status_report(format: str = "text") -> None:
        """Print per-framework version + last_verified + source_url to stdout.
        Used by `quirk compliance status` (D-05, COMPLY-09)."""
        seen: Dict[str, Dict[str, Any]] = {}
        for entries in COMPLIANCE_MAP.values():
            for e in entries:
                key = e["framework"]
                # Keep the OLDEST last_verified per framework (worst-case staleness signal).
                if key not in seen or e["last_verified"] < seen[key]["last_verified"]:
                    seen[key] = {
                        "framework": e["framework"], "version": e["version"],
                        "last_verified": e["last_verified"], "source_url": e["source_url"],
                    }
        if format == "json":
            import json as _json
            print(_json.dumps(seen, indent=2, sort_keys=True))
            return
        print(f"{'Framework':<20} {'Version':<14} {'Last Verified':<14} Source URL")
        print("-" * 100)
        for fw in sorted(seen):
            row = seen[fw]
            print(f"{row['framework']:<20} {row['version']:<14} {row['last_verified']:<14} {row['source_url']}")
