    [Net-new content per D-02 — covers four areas. ~6–10 short sub-sections; bullet style. Be specific.]

    ### 5.1 Scan failures

    - **Permission denied on a target** — confirm the QUIRK host can reach the port; check firewall/security-group rules.
    - **Timeouts** — adjust `scan.timeouts.<scanner>_seconds` blocks. See [`docs/timeout-retry-audit.md`](timeout-retry-audit.md) for per-scanner defaults.
    - **`missing_extra` advisory finding** — install the named extra (e.g., `pip install quirk[identity]` for Kerberos). Phase 45 INSTALL-02 surfaces these instead of silent skips.
    - **TLS handshake errors against modern endpoints** — confirm `cryptography` package version (Phase 45-01 D-07: do not let `[identity]` impacket downgrade it).

    ### 5.2 Database / output

    - **`db_path` permission error** — confirm the directory is writable. Default is `./quirk.db`.
    - **Migrations** — additive only (`_ensure_*_columns` in `quirk/db.py`); deleting `quirk.db` is safe but loses scan history.
    - **CBOM file generation** — emits `cbom-<ts>.json` and `.xml`; both must validate against CycloneDX 1.6.
    - **PDF render failure** — install `quirk[dashboard]` (includes Playwright); run `playwright install chromium` once on the host.

    - **Vite build errors** — only relevant when rebuilding the React SPA; the published wheel ships a built bundle at `quirk/dashboard/static/`.
    - **Stale `.vite/`** — delete `.vite/` in `src/dashboard/` and rebuild.
    - **Port conflict on 8512** — pass `quirk serve --port <other>`; loopback only by default.
    - **Data not loading** — confirm a recent scan has populated `quirk.db`; the dashboard reads SQLite directly.
