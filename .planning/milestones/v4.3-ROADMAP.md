## Milestone Summary

**Decimal Phases:**

None — all phases were planned at roadmap creation time.

**Key Decisions:**

- Phase 25 RS-family OIDC routing: `_derive_identity_findings()` in `scan.py` handles RS256/RS384/RS512 via `OIDC_ALG_SEVERITY` lookup; TLS-bleed guard prevents double-counting
- Phase 26 GCP creds: `DefaultCredentialsError` fires at API call time — `GCP_AVAILABLE` flag alone does not protect against it; must be caught explicitly
- Phase 27 dar_ prefix: 5th subscore prefix introduced parallel to `identity_` — keeps surface scoring separable for future per-surface dashboard breakdowns
- Phase 27 CRITICAL PATH: `_ensure_v43_columns()` and `dat_scan_json` column are shared dependencies for Phases 28, 29, 30; Phase 27 must complete before any of them
- Phase 28 ThreadPoolExecutor: S3 `list_buckets` is NOT paginated — `get_paginator('list_buckets')` raises `OperationNotPageableError`; use `ThreadPoolExecutor(max_workers=10)` for per-bucket calls
- Phase 28 GCS sentinel reuse: Phase 26 GCS bucket data reused in Phase 28 — zero duplicate `storage.buckets.list` API calls in a single scan run
- Phase 29 K8S-03 invariant: etcd `EncryptionConfiguration` is NOT a queryable K8s API resource — use managed cluster APIs or kube-apiserver pod spec; `encryption-config-inaccessible` finding is a required deliverable, not an error
- Phase 30 VAULT CBOM: Pass 1 unmodified — transit key algorithm registration goes through default else clause (D-14); Pass 2+3 skip `VAULT` for cert/protocol components
- Phase 31 trend analysis: `scanned_at`-based session grouping from `list_scans()` — no new SQLite table required; NULL collision with v4.2-era sessions is expected and documented
