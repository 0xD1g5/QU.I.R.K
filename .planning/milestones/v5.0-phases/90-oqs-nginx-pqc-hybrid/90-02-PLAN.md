<notes>
- **Security-critical (security gate):** the probe shells to `openssl`. Build the command as an argv LIST (`["openssl","s_client","-connect", f"{host}:{port}", "-groups","X25519MLKEM768",...]`), NEVER `shell=True` and never string-interpolate the target into a shell line. Validate/normalize host and int(port). Set a hard subprocess timeout (e.g. 8s) and feed `</dev/null` (stdin closed) so a hung handshake cannot stall the scan; on timeout, treat as "not detected", not a crash.
- Capability gate (D-01 discretion): before probing, detect host OpenSSL ML-KEM support — e.g. parse `openssl list -kem-algorithms` for an ML-KEM/X25519MLKEM768 entry, or feature-probe. If unsupported → advisory path.
- Do NOT use Python stdlib `ssl.set_ecdh_curve()` — it rejects the hybrid name even on OpenSSL 3.6 (spike-confirmed).
- Discriminator (spike-verified, false-positive-free): offering ONLY X25519MLKEM768 succeeds against the OQS server, fails against classical `tls-modern:443`. Parse `Negotiated TLS1.3 group: X25519MLKEM768` to confirm.
- Advisory path: emit a clearly-scoped ADVISORY finding/endpoint (mirror the existing `protocol="ADVISORY"` shape in quirk/cbom/writer.py) documenting "full PQC-hybrid detection needs host OpenSSL >= 3.5 / OQS-compiled tooling", AND still increment pqc_hybrid_endpoint_count (D-05) when the target is a configured PQC target.
- Helper location is Claude's discretion (D-01): new `quirk/scanner/pqc_probe.py` keeps it cleanly out of the sslyze path — preferred.
</notes>
