## What changed

### 1. Module-level `@pytest.fixture(autouse=True)` `_k8s_available_flags`

Patches:
- `quirk.scanner.k8s_connector.K8S_AVAILABLE → True`
- `quirk.scanner.k8s_connector.AKS_AVAILABLE → True`
- `quirk.scanner.k8s_connector.GKE_AVAILABLE → True`
- Adds `sys.modules["google"]` and `sys.modules["google.cloud"]` stubs *only if not already present* (parent-package stubs so `pkgutil.resolve_name` can traverse the dotted path used by `patch("google.cloud.container_v1.X", create=True)`).

Critical detail: leaves `sys.modules["google.cloud.container_v1"]` untouched. The production code in `quirk/scanner/k8s_connector.py:54-66` self-registers a `types.ModuleType` stub there at import time and binds it to a module-level alias `_gke_container`. Overwriting it in the fixture would break the link — `patch` would target a different MagicMock than the one `_scan_gke_encryption` actually uses. Leaving it alone means the patch and the production-code stub are the same object.

### 2. Fix `ApiException` fallback class signature

`test_secret_rbac_403_produces_insufficient_privileges` falls back to a synthesized `ApiException` class when `kubernetes.client.rest.ApiException` is unavailable. Old form used `type("ApiException", (Exception,), {...})` which inherits `Exception.__init__` (no kwargs accepted) — the test then called `ApiException(status=403, reason="Forbidden")` and got `TypeError`. Fixed by promoting the fallback to a proper class with an `__init__(self, status=None, reason=None, *args, **kwargs)`.
