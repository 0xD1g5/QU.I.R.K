DNSKEY algorithm minimum public-key byte lengths (RFC reference):
- RSA (algs 5, 7, 8, 10): exponent-length-prefix byte + 1-byte exp (min 1) + modulus (min 64 bytes for RSA-512, RFC 4034 §2.1.5) — practical floor: 5 bytes for the encoding header.
- ECDSA P-256 (alg 13): 64 bytes (RFC 6605)
- ECDSA P-384 (alg 14): 96 bytes (RFC 6605)
- Ed25519 (alg 15): 32 bytes (RFC 8080)
- Ed448 (alg 16): 57 bytes (RFC 8080)
Researcher: confirm the existing _parse_dnskeys algorithm dispatch table and use those constants.
