<objective>
Close TD-02 (96-REVIEW WR-03) per locked decision D-06. The REST fuzzer's
cascade counter currently resets to 0 on a request/connection exception
(rest_fuzzer.py:603), so a server failing via timeouts/connection resets never
accumulates the consecutive failures needed to trigger the cascade pause —
timeout-only servers escape the back-off entirely.

Purpose: A scan against an unresponsive host must observe the back-off pause
activate. Connection-level failures are at least as strong a "back off" signal
as 5xx responses.
Output: A combined failure-cascade counter (5xx + request exceptions) that
resets only on a genuine success response, plus a regression test.
</objective>
