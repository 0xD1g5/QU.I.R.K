<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fuzzer → target | Every dispatched URL crosses the scope/SSRF boundary; validate_external_url gates each |
| credential → finding record | Bearer / forged token must never enter stored CryptoEndpoint fields |
| target response → probe verdict | Untrusted server responses drive findings; must not be trusted for control flow beyond status/headers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-96-04 | Spoofing | per-request URL (SSRF) | mitigate | validate_external_url called before every session.request; internal/metadata IPs blocked; rejected URLs skipped without budget |
| T-96-05 | Denial of Service | runaway dispatch | mitigate | _resolve_budget hard cap (500) + budget_used break + TokenBucket 5 req/s + 3x5xx cascade pause |
| T-96-06 | Information Disclosure | bearer/forged token leak | mitigate | Phase 93 safe_str scrubbing; no token field on CryptoEndpoint; tested for absence |
| T-96-07 | Information Disclosure | alg-confusion HMAC secret | accept | The HMAC secret is the target's PUBLIC key — public by definition; no disclosure risk (RESEARCH security domain) |
| T-96-08 | Spoofing | jku/iss key-fetch SSRF | mitigate | Probe never follows jku/iss; only fetches /.well-known/jwks.json on the in-scope base_url, still scope-gated |
</threat_model>
