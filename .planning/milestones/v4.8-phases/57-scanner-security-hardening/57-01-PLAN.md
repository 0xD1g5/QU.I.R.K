<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → scanner | Operator-supplied target list / repo path / image ref crosses into network fetcher / subprocess invocation. Untrusted because target lists may originate from third-party inventories or attacker-controlled feeds. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-57-01 | Information Disclosure | url_allowlist.validate_external_url | mitigate | Reject RFC1918, link-local, loopback, file://, and metadata-service IPs by default; metadata IPs blocked even with allow_internal=True (cloud SSRF cred-theft chain). Closes CR-04 + CR-06 prerequisite. |
| T-57-02 | Tampering / Elevation | subprocess_input.validate_repo_path | mitigate | Reject leading dash (argv injection of `--config=evil`), shell metacharacters, `..`, and non-existent paths before subprocess.run. Closes CR-02 prerequisite. |
| T-57-03 | Tampering / Elevation | subprocess_input.validate_image_ref | mitigate | Reject leading dash, `dir:/`, `file://`, and shell metacharacters; require OCI-style ref regex. Closes CR-03 prerequisite. |

**Residual risks (documented, not mitigated in this phase):**
- DNS rebinding: hostname targets are accepted without DNS resolution. The
  CR-04 audit explicitly leaves "DNS-resolution guard for paranoia mode" out of
  scope. Acceptable because the broader threat model assumes operator-curated
  target lists; rebinding requires both attacker-controlled DNS and a victim
  pointing the scanner at the attacker domain.
- IPv6 representation aliases (`::ffff:10.0.0.5`, IPv4-in-IPv6) — `ipaddress.ip_address`
  parses these correctly into mapped IPv4 addresses, but operators bypassing
  via unusual encodings (`%2e%2e%2f`) are not in scope; URL parser does NOT
  pre-percent-decode hostnames per RFC 3986.

</threat_model>
