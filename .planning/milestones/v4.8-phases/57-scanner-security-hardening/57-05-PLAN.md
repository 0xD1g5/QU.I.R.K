```python
# semgrep

    [exe, "-o", "json", "--", image_ref],
    ...
)
```

    ```python
    from unittest.mock import patch, MagicMock
    import pytest

    from quirk.scanner.source_scanner import scan_source_repo


    @pytest.mark.parametrize("bad_path,expected_reason", [
        ("../../etc/passwd", "path_traversal"),
        ("/tmp/x; rm -rf /", "shell_metachar"),
        ("/tmp/$(whoami)", "shell_metachar"),
        ("--config=https://evil/rules.yml", "leading_dash"),
        ("-config=evil", "leading_dash"),
        ("", "path_traversal"),
    ])

             patch("shutil.which", return_value="/usr/bin/semgrep"):

            assert ep.scan_error_category == "invalid_input"

             patch("shutil.which", return_value="/usr/bin/semgrep"):
            endpoints = scan_source_repo("/no/such/path/abc123xyz", timeout=120)
            mock_run.assert_not_called()
            assert endpoints[0].scan_error == "nonexistent_path"
    ```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator target list → source/container scanner | repo_path / image_ref values originate from operator config and may be attacker-supplied via target lists. |
| scanner argv → semgrep / syft | argv values control the external scanner's behavior, including config loading and output destinations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-57-13 | Tampering / Elevation | scan_source_repo subprocess.run | mitigate | validate_repo_path rejects leading-dash, shell metachars, `..`, and non-existent paths BEFORE subprocess.run. Argv `--` separator added as defense in depth. Closes CR-02. |
| T-57-14 | Tampering / Elevation | scan_container_image subprocess.run | mitigate | validate_image_ref rejects leading-dash, `dir:/`, `file://`, shell metachars, and non-OCI-shaped refs. Argv `--` separator added. Closes CR-03. |
| T-57-15 | Information Disclosure | rejection-row CryptoEndpoint.host | mitigate | host stores ≤32-char redacted preview (control chars stripped) per D-08; preserves audit trail without echoing full attack payloads to logs / DB. |

**Residual risks:**
- semgrep / syft binary integrity is assumed. If `shutil.which` returns a
  trojaned binary, validation does not help. Out of scope (system-integrity
  concern, not scanner concern).
- The OCI-style image_ref regex is conservative — legitimate edge-case refs
  with unusual characters may be rejected. Acceptable: operators can rename
  the image or use a digest form. Documented in the helper's docstring (plan
  57-01).

</threat_model>
