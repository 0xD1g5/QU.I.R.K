<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test harness → app routes | TestClient simulates untrusted HTTP clients |
| Route introspection → app.routes | Meta-test reads FastAPI route dependency graph |
| Browser → CORS preflight | Origin header validated against allowlist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-04-E | Elevation of Privilege | New route added without auth | mitigate | D-06 route-introspection test fails CI if any POST/PUT/DELETE/PATCH lacks require_auth — regression gate |
| T-58-04-S | Spoofing | Test fixture leaks real QUIRK_API_TOKEN | mitigate | Tests use monkeypatch.setenv("QUIRK_API_TOKEN", "test-token") — isolated env, never touches real config |
| T-58-04-I | Information Disclosure | 401 vs 403 leaks auth state | accept | 401=no/bad token, 403=missing CSRF: this distinction is intentional per D-04/D-07; no sensitive data leaked |
| T-58-04-S | Spoofing | CORS Origin forgery from non-browser client | accept | CORS is browser-enforced; non-browser clients with valid bearer token are explicitly trusted; defense-in-depth via auth |
</threat_model>
