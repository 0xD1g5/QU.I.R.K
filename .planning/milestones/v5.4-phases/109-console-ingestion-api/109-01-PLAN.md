<objective>
Add the console-side provisioning seam — `quirk console enroll` — that writes a `sensors`
row and a `sensor_tokens` row (SHA-256 hash of a freshly minted bearer token) to the
console DB, and prints the raw bearer token once. This resolves the FLAGGED provisioning
gap from CONTEXT.md: `quirk sensor enroll` (Phase 108) is 100% local and never writes a
`sensors` row console-side, so without this command the locked "unknown sensor_id → 4xx"
rule (Plan 02) would make every push fail. This is provisioning, NOT part of the push
endpoint — consistent with the locked "provisioning out of push scope" decision.

Purpose: Make the push endpoint's sensor_id lookup coherent end-to-end (unblocks Phases 110/112).
Output: A new `enroll` sub-command in `quirk/cli/console_cmd.py`.
</objective>
