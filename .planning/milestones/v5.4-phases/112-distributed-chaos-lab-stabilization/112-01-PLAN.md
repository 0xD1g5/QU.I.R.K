<objective>
Build the distributed multi-segment chaos-lab topology that physically reproduces the
same-host:port-in-two-segments scenario (MERGE-03) under real Docker networking, plus the
lab.sh `distributed` command and a static topology pytest test as the automated verification floor.

The same-host:port reproduction uses a shared Docker DNS alias (`crypto.internal`): each segment's
TLS target carries the alias on its OWN segment network, and both sensors are configured to scan
`crypto.internal:443`. Each sensor's per-network Docker DNS resolves `crypto.internal` to ITS OWN
segment's reachable target — so both scans succeed AND both record the identical host:port string,
differing only by sensor_id/segment.

Purpose: Deliver everything runnable for the live distributed E2E (LAB-01/02), kept drift-free
per the CLAUDE.md chaos-lab rule (LAB-03). The live enroll→push→merge run is human-UAT (Plan handles
the runnable artifacts + automated static floor).
Output: docker-compose.distributed.yml, sensor.Dockerfile, distributed-e2e.sh, a `distributed` arm
in lab.sh, and tests/test_distributed_topology.py.
