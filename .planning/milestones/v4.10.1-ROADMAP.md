# v4.10.1 — Scoring Correctness Hotfix (SHIPPED 2026-05-22)

**Tagline:** Patch the marquee overall-readiness score so it stops displaying `100 / EXCELLENT` when subscores are red and CRITICAL findings are present — surgical normalization of the backend aggregator and the dashboard `ScoreGauge`, with the underlying penalty model untouched.

## Overview

v4.10 shipped with a triple-layer scoring bug: the overall readiness number always rendered `100 / EXCELLENT` regardless of actual posture. The root cause spanned backend aggregation and frontend gauge math, so a fix to only one half would have produced a *different* wrong number. v4.10.1 is a single-phase vertical MVP slice that fixes backend math, frontend gauges, and release engineering as one atomic, physics-coupled unit.

**Canonical case:** subscores `25 + 25 + 23 + 3 + 25 + 19 = 120` → before: **100 EXCELLENT** / after: **80 GOOD**.

### Plan breakdown

- **86-01 — Backend normalization (commit `7d0f71e`):** Replaced `_clamp(sum, 0, 100)` with `int(round(sum / 1.5))` at `quirk/intelligence/scoring.py`. Rewrote the module docstring to remove the misleading "Phase 60 SCORE-04 clamp-to-[0,100] is intentional" language. Added boundary tests asserting overall = 100 only when all six subscores hit their 25 ceiling, and 0 only when all are 0. `SCORE_WEIGHTS` and `_apply_weighted_impacts` left unchanged (0–25 subscore budget preserved). TDD RED→GREEN.
- **86-02 — Dashboard gauge fix (commits `fad2ced`, `620e5db`, `b9bab9e`):** Added `maxValue?: number` prop (default 100) to `ScoreGauge.tsx`; rewrote `_gaugeColor()` to operate on a normalized 0–1 fraction (red < 50 %, amber 50–79 %, green ≥ 80 %). Wired `maxValue={25}` into all six executive-page subscore radials and the standalone Data at Rest tab gauge. Vitest coverage for color + scale.
- **86-03 — Release engineering + operator UAT (commits `94ac361`, `34eb1f9`, `bce265b`):** Bumped `pyproject.toml [project.version]` 4.10.0 → 4.10.1 (version single-source-of-truth invariant preserved); towncrier changelog fragment in operator language with the canonical before/after example; authored HUMAN-UAT.md operator walkthrough — closed **PASS** on all four pass criteria after a hard browser refresh.
