### New Findings

#### INT-01: Dashboard scoring profile not propagated (Low-Medium)

`quirk/dashboard/api/routes/scan.py` line 330 calls `compute_readiness_score(evidence)` without
`profile=` kwarg. The function defaults to `balanced`. Users running a scan with `profile: strict`
will see different scores in the CLI report vs the dashboard.

- **Impact on default flow:** None (both default to `balanced`)
- **Impact on non-default profiles:** Score divergence between CLI report and dashboard
- **Fix path:** Store scan-time profile in DB, or have dashboard read `intelligence-*.json` artifact
- **Affected:** UI-02, CORE-01

#### INT-02: scorecard.py orphaned production module (Low)

`quirk/reports/scorecard.py::build_scorecard_markdown()` is never called in the production path.
`writer.py` uses its own `_scorecard_markdown()` inline. The module is only imported in tests.
It also lacks a `profile=` kwarg, creating a third stale scoring call site.

- **Impact on runtime:** None (never reached in production)
- **Fix path:** Delete `scorecard.py` or align with production path
- **Affected:** CORE-01 (single scoring path claim)
