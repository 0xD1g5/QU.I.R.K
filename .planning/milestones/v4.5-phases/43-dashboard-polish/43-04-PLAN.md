    Create `.github/workflows/dashboard-quality.yml`. The repo currently has no `.github/workflows/` directory (per D-19 + PATTERNS.md), so creating the directory + file is part of this task.

    Workflow content:
    ```yaml
    name: Dashboard Quality

    on:
      pull_request:
        paths:
          - 'src/dashboard/**'
          - '.github/workflows/dashboard-quality.yml'

    jobs:
      a11y:
        name: Axe + Console Gate
        runs-on: ubuntu-latest
        defaults:
          run:
            working-directory: src/dashboard
        steps:
          - uses: actions/checkout@v4

          - name: Setup Node
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'
              cache-dependency-path: src/dashboard/package-lock.json

          - name: Install dependencies
            run: npm ci

          - name: Build dashboard
            run: npm run build

          - name: Lint
            run: npm run lint

          - name: Run axe + console sweep (happy fixture)
            run: npm run a11y:check
            env:
              PUPPETEER_EXECUTABLE_PATH: /usr/bin/google-chrome-stable

          - name: Run axe + console sweep (empty fixture)
            run: npm run a11y:check:empty
            env:
              PUPPETEER_EXECUTABLE_PATH: /usr/bin/google-chrome-stable
    ```

    Notes:
    - `paths:` filter ensures the workflow only runs when dashboard files change (per D-19 rationale: keep this gate scoped, do not Node-tax every Python PR).
    - Ubuntu 22.04+ runners ship `google-chrome-stable` at `/usr/bin/google-chrome-stable`; setting `PUPPETEER_EXECUTABLE_PATH` makes puppeteer-core resolve it without depending on `channel: 'chrome'` (which can fail on some runner images). The harness in Plan 01 was written to fall back to `process.env.PUPPETEER_EXECUTABLE_PATH` if `channel: 'chrome'` doesn't resolve.
    - `npm ci` (not `npm install`) ensures the locked dep tree from package-lock.json is used.

    Decision implemented: D-19 (separate workflow at `.github/workflows/dashboard-quality.yml` triggered on src/dashboard/** PR paths).
