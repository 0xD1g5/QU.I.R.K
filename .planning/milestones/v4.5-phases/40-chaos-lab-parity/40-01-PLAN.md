    ```bash
    # Derive ALL profiles from docker-compose.yml. Output: alphabetized, deduped, one per line.
    # Preserves set -euo pipefail (no unbound vars; pipefail-safe — sort handles empty input).
    _derive_all_profiles() {
      if command -v yq >/dev/null 2>&1; then
        yq eval '.. | select(has("profiles")) | .profiles[]' "${COMPOSE_FILE}" 2>/dev/null \
          | sort -u
      else
        # Fallback: handles inline-array form (the only form in docker-compose.yml today).
        # Restricted character class [a-z0-9_-] mitigates shell-injection risk on parsed names.
        grep -E '^[[:space:]]*profiles:[[:space:]]*\[' "${COMPOSE_FILE}" \
          | grep -oE '"[a-z0-9_-]+"' \
          | tr -d '"' \
          | sort -u
      fi
    }
    ```

    all)
      mapfile -t _profiles < <(_derive_all_profiles)
      if [[ ${#_profiles[@]} -eq 0 ]]; then
        echo "❌ Could not derive profiles from ${COMPOSE_FILE}" >&2
        exit 1
      fi
      ALL_PROFILES=""
      for p in "${_profiles[@]}"; do ALL_PROFILES+=" --profile $p"; done
      echo "🔥 Starting ALL profiles: project=${PROJECT_NAME} file=${COMPOSE_FILE}"
      echo "   Profiles: ${_profiles[*]}"
      PROFILE_ARGS="${ALL_PROFILES}" compose up -d

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| docker-compose.yml → bash parser | YAML content is parsed by grep/sed and passed to docker compose as `--profile <name>` arguments. The compose file is repo-owned and not user-influenced at runtime, but the parsed output expands into shell argv. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01-01 | Tampering | `_derive_all_profiles()` grep parser | mitigate | Restrict character class to `[a-z0-9_-]` via `grep -oE '"[a-z0-9_-]+"'` so any malicious YAML edit cannot inject shell metacharacters (`;`, `$`, backtick, space) into the `--profile <name>` argv expansion. |
| T-40-01-02 | Elevation of Privilege | `yq` external dependency | accept | `yq` is invoked only if present (`command -v yq` guard); if compromised it could emit arbitrary names but those still pass through the same `sort -u` and only become docker compose `--profile` args. Same trust boundary as docker compose itself, which the operator is already trusting. |
| T-40-01-03 | Denial of Service | Parser empty-result handling | mitigate | `all)` arm checks `${#_profiles[@]} -eq 0` and exits 1 with a clear message rather than silently starting an empty lab. |
| T-40-01-04 | Information Disclosure | `./lab.sh profiles` output | accept | Profile names are public repo content (docker-compose.yml is committed); listing them disclosures nothing not already in the repo. |
</threat_model>
