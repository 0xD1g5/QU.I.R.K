## Overview

Expanded QU.I.R.K.'s cryptographic inventory surface to cover identity protocols — Kerberos, SAML/OIDC, and DNSSEC — each with a scanner module, CBOM integration, chaos lab profile, and a dedicated Identity tab in the dashboard. Three gap-closure phases (22, 23, 24) closed runtime bugs found by the milestone audit before shipping.

### Phase 18: DNSSEC Scanner

**Goal**: QU.I.R.K. can audit the DNSSEC posture of any domain — detecting unsigned zones, weak signing algorithms, NSEC zone enumeration exposure, and broken DS chains — with results in the CBOM
**Depends on**: Phase 17
**Requirements**: DNSSEC-01, DNSSEC-02, DNSSEC-03, DNSSEC-04, DNSSEC-05, DNSSEC-06, DNSSEC-07
**Plans**: 2 plans
**Completed**: 2026-04-09

Plans:
- [x] 18-01-PLAN.md — TDD test scaffold (RED tests for DNSSEC-01 through DNSSEC-07)
- [x] 18-02-PLAN.md — dnspython scanner, RFC 8624 classifier, CBOM integration, BIND9 chaos lab

**Summary**: Full DNSSEC scanner with authoritative NS resolution, DO-bit queries, RFC 8624/9905 algorithm classification (CRITICAL/HIGH/SAFE), NSEC/DS-chain detection, CBOM integration, and BIND9 chaos lab with 4 pre-signed zones (RSASHA1-weak, ECDSAP256SHA256-safe, broken-chain, unsigned).

**Decisions:**
- Direct authoritative NS query required for DNSSEC — system resolver strips DO bit and DNSKEY records
- DNSSEC_ALG_MAP placed in dnssec_scanner.py for scanner self-containment per RFC 8624/9905 3-tier
- CryptoPrimitive.PKE for RSA DNSSEC algorithms, SIGNATURE for DSA/ECDSA — CryptoPrimitive.RSA does not exist in cyclonedx
- Synthetic DNSSEC finding types (NONE/NSEC/DS-MISMATCH/SHA1-DS) excluded from CBOM algorithm registration

**Key Accomplishments:**
1. Three new identity protocol scanners — DNSSEC (RFC 8624/9905), SAML/OIDC (defusedxml XXE-safe), Kerberos (impacket AS-REQ) — expanding cryptographic inventory to the identity surface
2. Three Docker Compose chaos lab profiles — BIND9 (4 DNSSEC zones), SimpleSAMLphp (RSA-1024), Samba DC (RC4-enabled) — providing testbeds for all three identity scanners
3. Full identity CBOM pipeline — all three protocols produce CycloneDX components via dedicated elif branches; Pass 2/3 skip lists prevent hollow X.509 artifacts

6. Scan-session timestamp isolation — ISSUE-3 root cause eliminated; shared session_start ensures all identity endpoints share one scanned_at timestamp

**Decisions:**
- Direct authoritative NS query required for DNSSEC — system resolver strips DO bit
- impacket in [identity] extras only — pyOpenSSL conflict risk prevents core placement
- SAML_NS dict constant required — lxml XPath produces empty results without explicit namespaces
- No-preauth Kerberos case → placeholder endpoint rather than empty list (preserves observability)

**Issues Resolved:**
- P0: main_logger NameError in run_scan.py (Phase 22)
