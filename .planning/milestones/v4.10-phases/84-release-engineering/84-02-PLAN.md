  <name>Task 1: Add towncrier dev dependency + [tool.towncrier] config to pyproject.toml</name>
  <files>pyproject.toml</files>
  <action>
    Edit `pyproject.toml`:
    1. Add a `dev` optional-dependencies group (if not already present) containing `towncrier>=24.7.0`. If `[project.optional-dependencies]` already has groups but no `dev`, add it; otherwise create the group.
    2. Append a `[tool.towncrier]` config block at the end of the file with these keys:
       - `package = "quirk"` (the version is read from `quirk.__version__` which now derives from pyproject — closes the loop)
       - `directory = "changelog.d"`
       - `filename = "CHANGELOG.md"`
       - `title_format = "## [{version}] - {project_date}"` (matches Keep-a-Changelog heading style already in CHANGELOG.md)
       - `start_string = "<!-- towncrier release notes start -->\n"` (towncrier inserts above this marker)
       - `[[tool.towncrier.type]]` entries for each of: `feature` (showcontent=true), `bugfix` (showcontent=true), `doc` (showcontent=true), `removal` (showcontent=true), `misc` (showcontent=false)
    3. Insert the `<!-- towncrier release notes start -->` marker into `CHANGELOG.md` immediately before the first version section (after the existing header/intro lines) so towncrier knows where to prepend.
  </action>
  <verify>
    <automated>grep -c '\[tool.towncrier\]' pyproject.toml | grep -q '^1$' &amp;&amp; grep -c 'towncrier release notes start' CHANGELOG.md | grep -q '^1$'</automated>
  </verify>
  <done>pyproject.toml has the full `[tool.towncrier]` block with five type sections; `towncrier>=24.7.0` is in the `dev` extras group; CHANGELOG.md has the start_string marker positioned correctly.</done>
