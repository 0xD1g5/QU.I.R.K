      * `alice.der` — RSA 1024-bit key, SHA-1 signature (`-sha1`), CN=alice
      * `bob.der`   — RSA 1024-bit key, SHA-256 signature (`-sha256`), CN=bob
      * `carol.der` — RSA 2048-bit key, SHA-256 signature (`-sha256`), CN=carol
      Set `notAfter` to a far-future date so all three certs are non-expired
      (expiry path is exercised by a separate unit-test mock in Plan 79-04, not
      by lab fixtures — keep lab signals deterministic).
    - Execute `regen.sh` once and commit the resulting `.der` files to git so the

    Step B — Write `quantum-chaos-enterprise-lab/smime/ldif/users.ldif` with
    RFC 2849 base64 encoding of each DER cert (per D-79-R4 — `userSMIMECertificate;binary::`
    syntax for LDIF). Layout (organizational units first, then users):
    ```
    dn: ou=people,dc=quirk,dc=lab
    objectClass: organizationalUnit
    ou: people

    dn: uid=alice,ou=people,dc=quirk,dc=lab
    objectClass: inetOrgPerson
    cn: alice
    sn: alice
    uid: alice
    userSMIMECertificate;binary:: <base64-of-alice.der>

    (repeat for bob, carol)
    ```
    Generate the base64 via `base64 -w0 < alice.der` (Linux) or `base64 -i alice.der`
    (macOS) — `regen.sh` can append/refresh the LDIF stanza for reproducibility.

    Place after the `ldaps:` block. Per D-79-UPDATE use

    Compose service skeleton:
    ```
    smime-openldap:
      image: bitnamilegacy/openldap:2.6.10-debian-12-r4
      profiles: ["smime"]
      environment:
        LDAP_ROOT: "dc=quirk,dc=lab"
        LDAP_ADMIN_USERNAME: "admin"
        LDAP_ADMIN_PASSWORD: "admin"
        LDAP_PORT_NUMBER: 389
      volumes:
        - ./smime/ldif:/ldif:ro
        - ./smime/certs:/smime-certs:ro
      ports:
        - "38900:389"
      restart: unless-stopped

    smime-seed:
      image: bitnamilegacy/openldap:2.6.10-debian-12-r4
      profiles: ["smime"]
      depends_on:
        smime-openldap:
          condition: service_started
      entrypoint: ["/bin/sh", "-c"]
      # `-c` = continue on already-exists errors. Idempotency per D-79-R2
    ```
