<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scanner output → sanitize_scanner_text | All adversary-controlled strings (CN/SAN, hostnames, banners, error messages) cross this boundary before any render call |
| sanitize_scanner_text → nh3.clean | Pre-stripped text passed to Rust-backed allowlist sanitizer (empty tag/attr allowlists) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-01 | Tampering | sanitize_scanner_text input — None/non-str | mitigate | Explicit None guard returns ""; str() coercion wrapped in try/except returning "" on failure |
| T-78-02 | Tampering | URL injection (`javascript:`, `data:`, etc.) | mitigate | `_URL_RE` strips all known dangerous schemes BEFORE nh3 sees the text (nh3 has no plain-text URL stripper) |
| T-78-03 | Tampering | HTML tag injection (`<script>`, `<img onerror>`) | mitigate | nh3.clean with `tags=set(), attributes={}` strips every tag, preserves text content |
| T-78-04 | Spoofing | Caller substitutes a different sanitizer | mitigate | Module docstring declares chokepoint invariant; Plan 05 AST gate enforces |
| T-78-05 | Denial of Service | Pathological input (very large strings, regex catastrophic backtracking) | accept | `_URL_RE` is non-backtracking (`\S+`); nh3 is Rust-backed; per-call cost microseconds per RESEARCH performance note |
</threat_model>
