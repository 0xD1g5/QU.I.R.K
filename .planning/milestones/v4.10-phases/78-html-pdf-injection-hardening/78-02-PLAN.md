<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scanner finding dict → Jinja render | All adversary-controlled strings reach the HTML template via `findings`, `endpoints`, `roadmap` dict keys |
| operator config → `<title>` / header | `org_name`, `report_owner`, `data_classification` are operator-supplied; previously interpolated unsanitized into `<title>` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-06 | Spoofing | PDF Title/Author metadata derived from operator-controlled `org_name` | mitigate | Constant `<title>` + `<meta name="author">` — no interpolation in metadata path (D-78 Area 3) |
| T-78-07 | Tampering | `<script>` payload in finding title/host/description reaches HTML | mitigate | `| sanitize` at every Cluster B site strips tags; autoescape escapes any residual metachars |
| T-78-08 | Tampering | `javascript:` URL in operator-supplied `source_url` (compliance row) | mitigate | `| sanitize` strips known dangerous schemes via `_URL_RE` |
| T-78-09 | Tampering | New template author adds `| safe` without paired `| sanitize` | mitigate | Plan 05 AST gate enforces; this plan keeps the count at zero |
</threat_model>
