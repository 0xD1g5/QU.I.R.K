<interfaces>
From Phase 84 D-02 (carried in CONTEXT.md): PyPI distribution name is `qu-i-r-k` (NOT `quirk`). The Homebrew formula MUST install `qu-i-r-k` from PyPI, even though the Homebrew formula filename, class, and brew-binary name remain `quirk`.

Naming summary (do not confuse these):
- **PyPI dist name:** `qu-i-r-k` (used in `pip install` and `url "https://files.pythonhosted.org/.../qu-i-r-k-X.Y.Z.tar.gz"`)
- **Brew formula filename:** `Formula/quirk.rb` (Homebrew convention: filename matches the binary)
- **Brew formula class:** `class Quirk < Formula`
- **Installed binary:** `quirk`
- **Brew install command:** `brew install 0xD1g5/quirk/quirk` (tap repo `homebrew-quirk` shortens to `0xD1g5/quirk`)

`pipx`-isolated pattern in Homebrew: depend on `python@3.11` + `pipx`, then in `install`, run `pipx install --pip-args="--target=#{libexec}" qu-i-r-k[all]==#{version}` — or more idiomatic for tap formulae, build a venv under `libexec` and link bins via `bin.install_symlink`. Either is acceptable; the LAUNCH-02 success criterion is "pipx-managed venv" so the venv-isolation approach is what matters.
</interfaces>
