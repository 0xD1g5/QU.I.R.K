## Integration Findings

### INT-01 (P0) — `main_logger` NameError Crashes All Identity Scanning

**Affected:** DNSSEC-04, SAML-05, KERB-04
**Location:** `run_scan.py` lines 467, 469, 480, 482, 493, 495

All pre-identity scanner calls in `run_scan.py` correctly use `logger=logger` (the local variable defined at line 162). The three identity scanner blocks were written using `main_logger` — a name that is never defined anywhere in the file. When `enable_dnssec`, `enable_saml`, or `enable_kerberos` is `True` and targets are configured, the scan raises `NameError: name 'main_logger' is not defined` and crashes.

The unit tests did not catch this because they mock `scan_dnssec_targets`, `scan_saml_targets`, and `scan_kerberos_targets` directly, bypassing the `run_scan.py` orchestration path. The fix is trivial: replace all 6 occurrences of `main_logger` with `logger`.

```
run_scan.py:467   logger=main_logger,    → logger=logger,
run_scan.py:469   main_logger.info(...)  → logger.info(...)
run_scan.py:480   logger=main_logger,    → logger=logger,
run_scan.py:482   main_logger.info(...)  → logger.info(...)
run_scan.py:493   logger=main_logger,    → logger=logger,
run_scan.py:495   main_logger.info(...)  → logger.info(...)
```

### INT-02 (P1) — CBOM Pass 3 Generates Spurious TLS Components for SAML/KERBEROS

**Affected:** SAML-05, KERB-04
**Location:** `quirk/cbom/builder.py` line 468

Pass 3 in `build_cbom()` generates `ProtocolProperties` components. Non-TLS/SSH protocols are excluded via a skip list:
```python
elif ep.protocol in ("JWT", "CONTAINER", "SOURCE", "AWS", "AZURE", "DNSSEC"):
    continue
```
`"SAML"` and `"KERBEROS"` are absent. These endpoints fall through to the `else` branch (line 473) and generate spurious `crypto/protocol/tls/{host}:{port}` CycloneDX components — incorrect output in every CBOM produced with identity findings present.

Fix: add `"SAML"` and `"KERBEROS"` to the skip list at line 468.
