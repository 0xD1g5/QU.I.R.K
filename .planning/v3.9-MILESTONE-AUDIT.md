## Integration Checker Findings

### Open Issues: 3

#### MISMATCH-01 — Quantum Safety Label Type Mismatch (Medium Severity) — Still Present
**File:** `quirk/dashboard/api/routes/scan.py` lines 143, 379
**Issue:** `quantum_safety_label()` expects `int | None` NIST level but receives string algorithm name at two call sites.
- Line 143: `qs = quantum_safety_label(ep.cert_pubkey_alg)` then `if qs in ("Vulnerable", "At Risk")` — comparison never matches raw enum `"quantum-vulnerable"`, so quantum-vulnerability findings for non-RSA cert algorithms are silently dropped from the findings tab
- Line 379: `return quantum_safety_label(algorithm)` — cert inventory receives raw `"quantum-vulnerable"` string instead of display label `"Vulnerable"`
- CBOM viewer (lines 180-181) uses the **correct** two-step pattern and is unaffected

**Fix (Phase 10):**
```python
# Replace single-step calls with:
_, nist_level, _ = classify_algorithm(alg)
label = quantum_safety_label(nist_level)
```
**Affected requirements:** CBOM-03, UI-03

---

#### PACKAGE-01 — Dashboard Static Assets Not Packaged (High Severity) — Still Present
**File:** `pyproject.toml` `[tool.setuptools.package-data]`
**Current:** `quirk = ["reports/templates/*.j2", "config_template.yaml"]`
**Issue:** `quirk/dashboard/static/` (built React bundle) not in package-data. Non-editable `pip install quirk[dashboard]` ships the Python backend but omits the React UI. `quirk serve` serves placeholder HTML for all routes.

**Fix (Phase 10):**
```toml
quirk = ["reports/templates/*.j2", "config_template.yaml", "dashboard/static/**/*"]
```
**Affected requirements:** UI-01, BRAND-04

#### MISSING-01 — Intelligence Block Absent from config_template.yaml (Low Severity) — Still Present
**File:** `quirk/config_template.yaml`
**Issue:** Phase 9 delivered `profile: strict/balanced/lenient` and `calibration_overrides` in `intelligence/scoring.py`. Both `writer.py` and `executive.py` are wired to pass these values. However, `config_template.yaml` has no `intelligence:` section — users who run `quirk init` have no discoverable path to the scoring profile knob. Runtime safe (defaults apply).

```yaml
# intelligence:

#   # calibration_overrides:  # fine-tune individual weight keys
```
**Affected requirements:** CORE-01 (discoverability), BRAND-04 (zero-to-scan UX)
