## Integration Checker Findings

### Critical: 2 Issues Found

#### MISMATCH-01 — Quantum Safety Label Type Mismatch (Medium Severity)
**File:** `quirk/dashboard/api/routes/scan.py` lines 143, 379
**Issue:** `quantum_safety_label(ep.cert_pubkey_alg)` passes a string algorithm name where the function expects `int | None` NIST level. Silently returns `"quantum-safe"` for all algorithms — the dashboard certificate inventory and finding-level quantum risk badges are semantically wrong for RSA, DSA, DH, and other non-ECDSA-P384+ algorithms.
**Fix:** Replace single-step calls with the two-step pattern already used correctly at lines 180–181:
```python
_, nist_level, _ = classify_algorithm(alg)
label = quantum_safety_label(nist_level)
```
**Affected requirements:** CBOM-03, UI-03

#### PACKAGE-01 — Dashboard Static Assets Not Packaged (High Severity for Distribution)
**File:** `pyproject.toml` `[tool.setuptools.package-data]`
**Issue:** `quirk/dashboard/static/` (built React bundle) not declared in package-data. A non-editable `pip install quirk[dashboard]` ships the Python backend but omits the React UI. `quirk serve` starts but serves 404 for all UI routes.
**Fix:** Add to pyproject.toml:
```toml
[tool.setuptools.package-data]
quirk = ["reports/templates/*.j2", "config_template.yaml", "dashboard/static/**/*"]
```
**Affected requirements:** UI-01, BRAND-04

### Low Severity: 1 Issue Found

#### MISSING-01 — Intelligence Block Absent from config_template.yaml
**Issue:** `quirk init` generates a `config.yaml` with no `intelligence:` section. Users cannot discover the `profile: strict/balanced/lenient` knob from the generated template. Runtime works fine (defaults apply via `config_from_dict`).
**Fix:** Add commented section to `quirk/config_template.yaml`:
```yaml
# intelligence:
#   profile: balanced  # strict | balanced | lenient — adjusts agility/identity score weights
#   # calibration_overrides:  # fine-tune individual weight keys
```
**Affected requirements:** CORE-01 (discoverability), BRAND-04 (zero-to-scan UX)
